# 撮合算法设计 v1.1

> C2C 交易匹配引擎
> 设计目标：让好行为获得流量，差行为悄悄失去流量；用户无感
> 不影响：现有的"一键买币 / 挂单广场"两大入口形态

---

## 0. 设计背景

### 现状（2024）
- 一键买币：随机/简单匹配
- 挂单广场：按时间倒序展示
- 没有基于信用的差异化分配
- 高信用商户的优势仅体现在"标识"，不体现在"流量"

### 目标（2025）
- **流量即奖惩**：高信用 → 多生意；低信用 → 少生意
- **撮合层透明可控**：所有规则可配置、可灰度、可回滚
- **用户无感**：买家点一键买币照样秒成交，体验无变化
- **抗 Sybil**：互相养号无法获得流量

---

## 1. 核心架构

```
   买家请求（一键买币 / 浏览挂单）
                ↓
    ┌────────────────────────┐
    │   候选池构建（Filter） │
    │   ─ 在线状态筛选         │
    │   ─ 价格区间筛选         │
    │   ─ 限额匹配           │
    │   ─ 黑名单 / 风控筛选    │
    │   ─ 多样性筛选         │
    └────────────────────────┘
                ↓
    ┌────────────────────────┐
    │   评分排序（Rank）     │
    │   ─ 信用分权重         │
    │   ─ 准时放币率权重     │
    │   ─ 实时行为权重       │
    │   ─ 多样性权重         │
    │   ─ 价格权重           │
    └────────────────────────┘
                ↓
    ┌────────────────────────┐
    │   分配执行（Dispatch）  │
    │   ─ Top-K 分发           │
    │   ─ 反 Sybil 校验        │
    │   ─ 冷却 / 限频           │
    │   ─ 失败 fallback        │
    └────────────────────────┘
                ↓
        匹配卖家 → 成单
                ↓
    ┌────────────────────────┐
    │   反馈学习（Learn）     │
    │   ─ 撮合成功率           │
    │   ─ 卖家放币时长         │
    │   ─ 工单率                │
    │   ─ 调整下次权重         │
    └────────────────────────┘
```

## 2. 候选池构建（Filter）

### 2.1 在线状态筛选 ⭐ 核心

```python
def is_online(seller):
    return (
        seller.app_active_within(minutes=15)
        OR seller.has_recent_activity(minutes=30)
    )
```

→ **不在线的卖家根本不进候选池**。
→ 这是解决"卖家睡觉不发货"的关键预防措施。

### 2.2 价格区间筛选

```
卖家挂单价 ∈ [当前市场价 ± 2%]
```

### 2.3 限额匹配

```
trade_amount >= seller.min_per_order
trade_amount <= seller.max_per_order
buyer.daily_remaining >= trade_amount
seller.daily_remaining >= trade_amount
```

### 2.4 黑名单 / 风控筛选

| 卖家被排除条件 |
|---|
| 永久停权 |
| 处于工单中（工单未结） |
| 24 小时内有 2 次以上工单 |
| 反洗钱命中 |
| 被买家加入黑名单 |
| 被平台标记为可疑 |

## 3. 评分排序（Rank）

### 3.1 综合得分公式

```
score = w1 * credit_score
      + w2 * on_time_rate_score
      + w3 * activity_score
      + w4 * diversity_bonus
      + w5 * price_advantage
      - w6 * recent_failure_penalty

权重默认值：
w1 = 0.30  (信用分)
w2 = 0.25  (准时放币率)
w3 = 0.15  (实时活跃度)
w4 = 0.10  (撮合多样性奖励)
w5 = 0.15  (价格优势)
w6 = 0.05  (近期失败惩罚)
```

### 3.2 各分项详解

#### credit_score（信用分）

```
直接取信用体系的 0–950 分，归一化到 0–1
```

#### on_time_rate_score（准时放币率）

```
取近 30 天准时放币率：
≥ 95%：1.0
85–95%：0.7
70–85%：0.4
< 70%：0.1
```

#### activity_score（实时活跃度）

```
近 5 分钟有 App 活动：1.0
近 15 分钟有：0.8
近 30 分钟有：0.5
近 60 分钟有：0.2
> 60 分钟：0
```

→ 鼓励商户保持在线，**比"通知"有效得多**。

#### diversity_bonus（撮合多样性）

```
买家与该卖家近 30 天交易次数：
0 次：1.0（新对手方加分）
1–2 次：0.7
3–5 次：0.3
6+ 次：0
```

→ 防止"老熟客"垄断流量。

#### price_advantage（价格优势）

```
卖家挂单价相对市场价：
低 0.5% 以上：1.0
-0.5% 到 +0.5%：0.5
+0.5% 以上：0.2
```

#### recent_failure_penalty（近期失败惩罚）

```
近 7 天工单数：
0：0
1：0.3
2：0.7
3+：1.0（实际上已被风控筛选掉）
```

### 3.3 等级权益的"硬性加成"

在公式之上，等级带来额外的流量倍率：

| 等级 | 流量倍率 |
|---|---|
| L5 | × 6.0 |
| L4 | × 3.0 |
| L3 | × 1.5 |
| L2 | × 1.0 |
| L1 | × 0.2* |

*(注：新注册 30 天内 L1 享受新手保底，详见主文档)*

```
final_score = score × level_multiplier
```

## 4. 分配执行（Dispatch）

### 4.1 一键买币模式

```python
def one_click_buy(buyer, amount):
    candidates = build_candidate_pool(buyer, amount)
    ranked = rank(candidates, buyer)

    # 取 Top 5，加权随机选 1 个
    top5 = ranked[:5]
    selected = weighted_random_choice(top5)

    if selected:
        return assign_order(buyer, selected, amount)
    else:
        return fallback_to_listing_square(buyer, amount)
```

> 使用 Top-5 加权随机而非"取最高"，是为了：
> - 避免少数 L5 商户垄断
> - 给中段商户机会
> - 降低被预测/被刷的风险

### 4.2 挂单广场模式

```python
def listing_square(buyer, filter_params):
    candidates = build_candidate_pool(buyer, filter_params)
    ranked = rank(candidates, buyer)

    # 不取 Top-K，全量返回但按分数排序
    return paginate(ranked, page_size=20)
```

→ 买家自由选择，但系统已经把"最值得展示"的排前面。

### 4.3 反 Sybil 校验（执行前最后一道）

```python
def anti_sybil_check(buyer, seller):
    if buyer.sybil_group_id == seller.sybil_group_id:
        log_suspicious_match(buyer, seller)
        return False

    if buyer.last_seller_id == seller.id and time_since < 60s:
        return False  # 防快速重复匹配同一对

    return True
```

### 4.4 冷却 / 限频

```
- 同一卖家 1 分钟内不会被一键买币匹配 ≥ 2 次
- 同一买家与同一卖家 24 小时内匹配次数 ≤ 5 次
- 卖家被工单后 1 小时内不参与一键买币
```

### 4.5 Fallback 机制

```
如果候选池为空或分配失败：
1. 放宽多样性筛选
2. 放宽在线状态筛选（扩到 60 分钟）
3. 仍无结果 → 引导买家进入挂单广场
4. 极端情况：通知运营介入（流动性告警）
```

## 5. 不同场景下的权重调整

### 5.1 大额交易（≥ 100,000 元）

```
w1 (信用分) = 0.40
w2 (准时率) = 0.30
其他相应降低
→ 大额更看重信用,降低风险
```

### 5.2 新人买家（注册 ≤ 7 天）

```
强制只匹配 L4+ 卖家
w1 (信用分) = 0.50
→ 保护新人,首次体验良好
```

### 5.3 凌晨时段（00:00–06:00）

```
w3 (活跃度) 权重 = 0.40
→ 凌晨在线本身就稀缺,大幅倾向活跃用户
→ 自然解决"睡觉商户"问题
```

### 5.4 高峰期（19:00–22:00）

```
w4 (多样性) 权重 = 0.20
→ 高峰期分散流量,防止头部商户撑爆
```

## 6. 商户视角的"流量公式"

为了让商户能自我优化，提供一个**简化版透明公式**（在商户后台展示）：

```
你的流量分 = 你的等级 × 准时率 × 活跃度 × 多样性

要提升流量,你可以：
1. 升级（→ 信用分+经验值）
2. 提升准时率（≥ 95% 拿满分）
3. 保持在线（每 15 分钟有活动）
4. 服务多元买家（不要只服务老熟客）
```

→ **商户能看懂、能行动、能感知**。这是行为引导的关键。

## 7. 灰度发布

### 7.1 灰度路径

```
T+0：新算法开发完成,影子模式运行
       → 跑分对比新旧算法决策差异

T+7：5% 流量灰度
       → 监控撮合成功率、平均放币时长、工单率

T+14：20% 流量灰度
       → 验证大额场景

T+30：50% 流量灰度
       → 对比关键指标

T+60：100% 全量
```

### 7.2 灰度回滚机制

任何指标恶化超过阈值,自动回滚：

```
撮合成功率下降 > 2pp → 回滚
平均放币时长上升 > 30s → 回滚
工单率上升 > 10% → 回滚
商户投诉率上升 > 5% → 回滚
```

## 9. 与信用体系的双向反馈

### 9.1 信用 → 撮合（流量分配）

见第 3 节。

### 9.2 撮合 → 信用（行为采集）

```
每次撮合行为都会回写信用体系：
- 一键买币成功匹配 → 卖家 +500 经验
- 卖家拒接 → 卖家 -1,000 经验
- 卖家放币时长 → 准时率统计更新
- 工单结果 → 工单率统计更新
- 多样性数据 → 多样性指数更新
```

→ 形成正循环：行为好 → 流量好 → 数据好 → 信用好 → 流量更好。

## 10. 风险与边界

### 10.1 流量集中风险

如果 L5 商户太少,可能集中度过高。

**对策**：
- Top-K 加权随机选择（非取最高）
- 每个商户单日最大撮合数封顶
- 监控 GINI 系数,过高时降低 level_multiplier 差距

### 10.2 新商户冷启动

新商户没有信用,流量差,可能进不来。

**对策**：
- 新商户首 30 天有"扶持流量"（保底匹配 5 单/天）
- 但金额受沙箱限制,风险可控

### 10.3 算法可被反推

商户可能逆向推测公式,集中刷某项指标。

**对策**：
- 公开"简化公式",但实际用"完整公式"
- 部分参数动态浮动（小幅）
- 反作弊检测异常行为模式
