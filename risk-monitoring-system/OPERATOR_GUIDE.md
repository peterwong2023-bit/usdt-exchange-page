# 高可用AI风险监控系统 - 运营人员使用指南

## 📌 快速开始

### 访问系统
打开浏览器访问: `http://localhost:8000/index-v2.html`

## 🎯 核心功能模块

### 1. 概览仪表板 (Overview Dashboard)

#### 1.1 顶部快速统计
- **紧急告警**: 需要立即处理的高危风险
- **高风险用户**: 当前标记为高风险的用户总数
- **疑似团伙**: 检测到的团伙数量
- **今日处理**: 今天已处理的风险案例数

#### 1.2 实时告警中心
**功能**: 显示最新的风险告警信息

**操作步骤**:
1. 查看告警列表，红色为紧急，橙色为高风险
2. 点击 ✓ 按钮快速标记为已处理
3. 点击告警卡片查看详细信息
4. 使用"全部已读"按钮批量标记

**告警颜色说明**:
- 🔴 红色边框: 紧急告警 (需立即处理)
- 🟠 橙色边框: 高风险告警 (需尽快处理)

#### 1.3 风险趋势分析
**功能**: 显示近7天/30天的风险趋势变化

**如何使用**:
- 通过下拉菜单切换时间范围 (24小时/7天/30天)
- 观察各风险等级的变化趋势
- 识别风险高峰时段

#### 1.4 Top 10 高风险用户
**功能**: 显示当前风险评分最高的10个用户

**操作步骤**:
1. 点击任意用户卡片查看详情
2. 金色数字表示前3名
3. 查看主要风险事件摘要

#### 1.5 团伙网络监控
**功能**: 显示检测到的疑似团伙

**操作步骤**:
1. 查看团伙成员数量和交易金额
2. 点击团伙卡片查看关系图谱
3. 查看关联的风险标签

---

### 2. 用户风险列表 (User Risk List)

#### 2.1 高级过滤
**可用过滤条件**:
- 风险等级: 紧急/高/中/低
- 风险类型: 设备/IP/支付/交易/团伙
- 处理状态: 待处理/处理中/已处理/已忽略
- 用户搜索: 输入用户ID/手机号/设备ID
- 日期范围: 选择起止日期

**操作步骤**:
1. 选择过滤条件
2. 点击"搜索"应用过滤
3. 点击"重置"清空所有条件
4. 点击"导出"下载Excel文件

#### 2.2 批量操作
**功能**: 对多个用户执行统一操作

**操作步骤**:
1. 勾选表格左侧复选框选择用户
2. 或点击表头复选框全选
3. 批量操作栏会自动显示
4. 选择操作: 批量限制/批量封禁/批量审核
5. 输入操作原因并确认

**批量操作类型**:
- **批量限制**: 限制用户的交易权限
- **批量封禁**: 完全封禁用户账户
- **批量审核**: 将用户加入人工审核队列

#### 2.3 单个用户操作
**操作步骤**:
1. 点击用户行的"详情"按钮
2. 在详情弹窗中查看完整信息
3. 选择操作: 限制交易/封禁用户/加入白名单
4. 填写操作原因（必填）
5. 确认执行

---

### 3. 风险事件监控 (Event Monitoring)

#### 3.1 事件统计卡片
快速查看各等级事件数量:
- 🔥 紧急事件 (红色)
- ⚠️ 高风险事件 (橙色)
- ℹ️ 中风险事件 (蓝色)
- ✓ 低风险事件 (绿色)

#### 3.2 事件时间线
**功能**: 按时间顺序显示所有风险事件

**如何阅读**:
- 时间线从上到下，最新在顶部
- 左侧圆点颜色表示风险等级
- 显示触发用户和自动处理动作

**自动刷新**:
1. 点击"自动刷新"按钮启动
2. 每5秒自动更新一次
3. 再次点击停止刷新

---

### 4. 关系图谱分析 (Relationship Graph)

#### 4.1 生成关系图
**操作步骤**:
1. 选择图谱类型 (设备/IP/支付/团伙/全关系)
2. 输入中心节点用户ID
3. 选择关系深度 (1-3层)
4. 点击"生成图谱"

#### 4.2 图谱解读
**节点类型**:
- 🔴 红色圆形: 高风险用户
- 🟠 橙色圆形: 中风险用户
- 🔵 蓝色方形: 设备节点
- 🟢 绿色菱形: IP节点
- 🟣 紫色三角: 支付节点

**操作技巧**:
- 鼠标滚轮: 缩放图谱
- 鼠标拖拽: 移动视图
- 点击节点: 查看节点详情
- 连线表示关联关系

---

### 5. 运营操作台 (Operations Panel)

#### 5.1 快速操作按钮
**6大快速操作**:
1. **封禁用户**: 快速封禁指定用户
2. **限制交易**: 临时限制交易权限
3. **解除限制**: 解除之前的限制
4. **人工审核**: 提交人工审核队列
5. **加入白名单**: 标记为可信用户
6. **加入黑名单**: 永久拉黑

**使用方法**:
1. 点击对应操作卡片
2. 输入用户ID
3. 填写操作原因
4. 确认执行

#### 5.2 待处理任务
**功能**: 显示需要人工处理的任务

**任务优先级**:
- 🔴 Urgent: 紧急任务 (立即处理)
- 🟠 High: 高优先级 (今日处理)
- 🟡 Medium: 中优先级 (本周处理)

**操作步骤**:
1. 点击任务卡片查看详情
2. 认领任务
3. 处理并记录结果
4. 标记任务完成

#### 5.3 最近操作记录
查看自己和团队的最近操作历史，包括:
- 操作类型和对象
- 操作原因
- 执行结果
- 操作时间和操作员

## 🚨 常见场景处理流程

### 场景1: 检测到疑似团伙欺诈

**处理流程**:
1. 在"概览仪表板"查看告警详情
2. 点击用户ID跳转到用户列表
3. 选中所有相关用户（勾选复选框）
4. 点击"批量限制"临时冻结交易
5. 切换到"关系图谱"生成团伙关系图
6. 分析关联设备、IP、支付方式
7. 切换到"运营操作台"创建审核任务
8. 指派给专人深度调查
9. 根据调查结果决定是否批量封禁

### 场景2: 单个用户异常交易

**处理流程**:
1. 在"实时告警"中发现异常交易提示
2. 点击用户ID查看详细信息
3. 检查用户的:
   - 风险评分和等级
   - 触发的风险事件
   - 最近交易记录
   - 设备和IP信息
4. 如果是误报: 点击"加入白名单"
5. 如果确认风险: 
   - 中低风险 → "限制交易"观察
   - 高风险 → "封禁用户"
6. 在操作原因中详细记录判断依据

### 场景3: 定期巡检

**每日巡检清单**:
1. 查看顶部统计数据，了解整体情况
2. 处理所有"紧急告警"
3. 查看"Top 10高风险用户"，关注新增用户
4. 检查"团伙网络监控"，处理新检测团伙
5. 查看"风险趋势"，识别异常波动
6. 处理"待处理任务"列表
7. 导出今日报表存档

## 💡 最佳实践建议

### 1. 风险处理优先级
```
紧急告警 > 团伙风险 > 高风险用户 > 中低风险用户
```

### 2. 操作原因填写规范
**必须包含**:
- 判断依据（如: 触发XX规则）
- 风险描述（如: 检测到设备共享）
- 处理理由（如: 疑似团伙欺诈）

**示例**:
```
用户触发多设备风险规则，24小时内使用5个不同设备登录，
且与已知欺诈团伙成员共享支付方式QR_889922，
疑似团伙作案，临时限制交易等待人工审核。
```

### 3. 避免误判
**检查清单**:
- ✅ 是否有多个风险指标同时触发？
- ✅ 风险评分是否持续上升？
- ✅ 是否有历史风险记录？
- ✅ 关系图谱是否显示可疑关联？
- ✅ 用户是否有申诉或解释？

**建议操作**:
- 单一风险指标 → 先观察或限制交易
- 多个风险指标 → 限制交易 + 人工审核
- 确认欺诈行为 → 立即封禁

### 4. 团队协作
- 使用"待处理任务"分配工作
- 在操作记录中@相关同事
- 定期开会讨论疑难案例
- 建立风险案例库

## 🆘 常见问题

**Q: 为什么有些用户显示为"处理中"但没有分配给我？**
A: 该用户可能已被其他运营人员认领。在"运营操作台"可以看到任务分配情况。

**Q: 如何撤销误操作？**
A: 使用"解除限制"功能，并在原因中说明是误操作撤销。所有操作都有日志记录。

**Q: 关系图谱加载很慢怎么办？**
A: 降低关系深度（从3层改为2层或1层），或选择特定图谱类型而不是"全关系图谱"。

**Q: 批量操作最多可以选择多少用户？**
A: 建议单次批量操作不超过50个用户，大批量操作建议分批进行。

**Q: 风险评分如何计算？**
A: AI系统综合考虑设备、IP、支付、交易、行为等多个维度，动态计算风险评分（0-100分）。
