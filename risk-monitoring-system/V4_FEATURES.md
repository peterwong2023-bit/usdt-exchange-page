## 🆕 V4 新增核心模块

### 1. KPI概览仪表板 📊

#### 5大关键指标
```
✅ 今日风险事件总数 (523)
✅ 高危事件数量 (12)  
✅ 新增黑名单用户 (4)
✅ 待处理事件队列 (47)
✅ 今日已处理数量 (156)
```

#### 特性
- **数字动画**: 数字从0缓动增长到目标值
- **趋势指示**: 显示相比昨天的涨跌百分比
  - 🔺 红色上箭头: 风险上升
  - 🔻 绿色下箭头: 风险下降
  - ➖ 灰色横杠: 保持稳定
- **颜色编码**: 
  - 🔴 危险红: 今日风险事件
  - 🔥 紧急红: 高危事件
  - 🟠 警告橙: 新增黑名单
  - 🔵 信息蓝: 待处理队列
  - 🟢 成功绿: 已处理数量

#### 数据来源
```javascript
kpiSummary = {
  total_risk_events_today: 523,
  high_risk_events: 12,
  medium_risk_events: 233,
  new_blacklisted_users: 4,
  resolved_today: 156,
  trend: {
    total_events: 12,    // +12%
    high_risk: -3,       // -3%
    new_blacklist: 0,    // 0%
    resolved: 15         // +15%
  }
}
```

### 2. AI模型质量监控 🤖

#### 4大核心指标
```
1. 命中准确率: 92%
   - 进度条可视化
   - 绿色表示健康 (>90%)
   - 橙色表示警告 (80-90%)
   - 红色表示异常 (<80%)

2. 误报率: 3%
   - 越低越好
   - 绿色健康 (<5%)
   - 目标: <3%

3. 人工干预次数: 47次/今日
   - 显示今日人工审核数量
   - 数字动画效果

4. 模型版本: v2.4.1
   - 显示当前运行的模型版本
   - 提供"查看更新"按钮
```

#### 为什么重要？
- **评估AI效果**: 准确率直接反映模型质量
- **优化方向**: 通过误报率找到改进空间
- **人力成本**: 人工干预次数越少，自动化程度越高
- **版本追踪**: 便于回溯问题和对比效果

#### 数据结构
```javascript
aiModelHealth = {
  ai_hit_rate: 92,              // 命中准确率 %
  false_positive_rate: 3,       // 误报率 %
  manual_override_count: 47,    // 人工干预次数
  model_version: "v2.4.1",      // 模型版本
  last_updated: "2025-11-27",   // 最后更新时间
  status: "normal"               // 运行状态
}
```

```

   - 红色: 异常
```

- 每个指标都有**状态灯**（绿色圆点 = 正常）
- 异常时会变为**橙色或红色**，并触发告警

### 8. 快速搜索功能 🔍

#### 支持搜索类型
```
1. 用户ID        (U开头)
2. 设备指纹      (DFP_或FP_开头)
3. 支付QR        (QR_开头)
4. 银行账号      (BANK_开头)
5. IP地址        (xxx.xxx.xxx.xxx格式)
```

#### 智能识别
系统自动识别搜索内容类型并跳转到相应页面：
- 用户ID → 用户风险列表
- 设备/支付/IP → 关系图谱分析

#### 位置
- 顶部导航栏中央
- 宽搜索框，支持回车快捷键
