## 16. 操作步骤描述（执行版）

> 用法：执行任意测试用例时，先按「16.1 通用执行流程」，再套用对应模块步骤模板。

### 16.1 通用执行流程（适用于全部用例）

1. **准备环境**
   - 打开目标环境（生产或测试域名），确认网络正常。
   - 清理缓存：必要时清除 `localStorage`、`sessionStorage`、Cookie。
   - 记录浏览器与设备信息（Chrome 版本、系统、分辨率）。
2. **进入页面**
   - 按用例中的页面路径进入目标页（如 `index.html` 或 `/lottery-admin/...`）。
   - 等待页面稳定（首屏渲染完成，骨架/加载状态消失）。
3. **执行操作**
   - 严格按用例“测试用例”字段操作（点击、输入、切换、刷新、回退等）。
   - 涉及时间逻辑的用例（倒计时/自动轮播）需完整观察一个周期。
4. **结果核对**
   - 对照“预期结果”逐条核验 UI、数据、状态、交互反馈。
   - 必要时检查 `localStorage`、URL 参数、控制台报错。
5. **记录结果**
   - 通过：标记 ✅ 并记录执行时间。
   - 失败：标记 ❌，附「实际结果 + 截图 + 复现步骤 + 浏览器信息」。

### 16.3 高风险场景专用步骤

#### A. 会话安全（AUTH-SEC-*）
1. 清除登录态，直接访问受保护页面（`user.html`、`me.html`、`/lottery-admin/...`）。
2. 观察是否被拦截并重定向到登录流程。
3. 登录后手动篡改 `localStorage` 中 token，刷新页面。
4. 观察是否失效并要求重新登录。
5. 多标签场景：A 页退出，B 页继续操作，确认会话同步失效。

#### B. 表单校验（ADMIN-VALID-*）
1. 打开对应创建/编辑弹窗。
2. 依次输入：空值、超长、非法字符、边界数值、反向时间区间。
3. 点击保存，验证是否阻止提交并给出明确错误提示。
4. 输入合法值再次提交，验证成功提示与列表更新。

#### C. 上传安全（UPLOAD-SEC-*）
1. 准备 4 类文件：合法图片、超大图片、非图片、双扩展名文件。
2. 逐个上传并记录系统反馈。
3. 验证仅合法图片可提交，其余应被拦截且有原因提示。
4. 模拟失败后重试，确认状态可恢复。

#### D. 错误处理（ERR-HANDLE-*）
1. 模拟接口超时/500/异常数据（可通过 Mock 或 DevTools）。
2. 观察页面是否出现错误态与重试入口。
3. 点击重试后验证是否可恢复正常。
4. 检查控制台是否存在未捕获异常。
