风险检测标准
系统通过 KYT (Know Your Transaction) 技术对每笔入款和出款目标地址进行风险评分，评分范围 0-100。评分越高，代表关联黑灰产的风险越大。商户可自定义风险评分阈值（当前 80 分），评分 ≥ 阈值的资金按当前模式隔离或分桶处理。
入款分桶规则
●干净桶：风险评分 < 80 的资金，无论标签如何均进入干净桶正常结算。
●风险桶：评分 ≥ 80 且命中商户配置的风险标签，资金隔离入桶，需手动处理。
↩退回：评分 ≥ 80 但标签未配置进风险桶，资金隔离，由付款方申请后退回。
⊘平台拦截：命中 S0 最高危标签（恐怖融资、受制裁实体等），直接拒绝入款并退回，不进入任何桶。
地址污染管理
退款
风险入款将被隔离，由付款方申请后退回。为防止持续恶意试探，系统将自动为用户派生并展示新的干净地址。
全收
入款地址收到风险资金后将被永久标记污染（后续资金自动进风险桶）。同时系统会自动派生新地址，保障后续充值不受影响。
API 出款路由规则
退款模式
安全地址 → 正常出款
风险/拦截地址 → 拒绝出款并提示更换
全收模式
安全地址 → 从干净桶出
风险地址 → 从风险桶出 (不足则拒绝)
拦截地址 → 直接拒绝
什么是"桶提款"（全收模式）？
风险桶支持在后台手动提到任意地址，无需审批。提款记录推送至工作群留痕。
高危名单包含哪些？
受制裁实体（OFAC 等）、恐怖融资、毒品、军火、儿童色情、执法查扣资产等。由平台统一维护，商户不可自定义。
