# AI 桌面便签 (AI Sticky) - 竞品分析

## 分析概要
目前桌面便签市场正处于从“纯文本容器”向“仪表盘化/AI化”转型的阶段。传统竞品（如 macOS Stickies）缺乏数据交互，专业仪表盘工具（如 Dashdot）上手门槛高。我们的机会在于将 **实时数据流（Crypto/Stock/Map）** 与 **AI 意图理解** 结合，通过“透明窗口+阈值告警”实现极客化的低吸纳成本信息中心，填补“轻量级盯盘+原子化 AI 助理”的市场空白。

## 竞品对比总览

| 维度 | AI Sticky (规划) | macOS Stickies | Binance PWA/Desktop | Raycast/Alfred | WidgetWall |
|------|-------------|-------------|-----|--------|-------|
| **功能完整度** | 高（数据+AI+记事） | 极低（仅文字） | 中（仅限行情） | 高（插件丰富） | 中（仅组件） |
| **用户体验** | 极简、透明置顶 | 原生、简单 | 重、侵占屏幕 | 需唤起、非长驻 | 桌面组件式 |
| **技术亮点** | 通用 DSL/本地运行 | 系统集成 | 实时 Websocket | 插件生态 | WebView 容器 |
| **差异化** | **数据驱动告警+AI 整理** | 免费内置 | 专业交易工具 | 快捷指令检索 | 纯视觉美化 |

## 各竞品详细分析

### macOS Stickies (原生便签)
- **功能描述**：经典的系统内置便签，支持简单的文本记录和颜色更改。
- **优点**：零成本，系统级稳定，极致轻量，支持置顶。
- **缺点**：死板，无任何自动化能力，无法接入 API，不支持 Markdown。
- **参考点**：其“半透明”及“浮动窗口”的物理交互模型是用户习惯的基准。

### Binance Desktop / OKX 行情窗口
- **功能描述**：交易所提供的悬浮窗功能，用于监控实时币价。
- **优点**：数据极准、毫秒级延迟，支持深度图。
- **缺点**：仅限币种，无法扩展天气、交通或笔记；样式极其“硬核”，不适合办公场景。
- **参考点**：其阈值报警（价格线颜色突变）的逻辑是硬核投资者的刚需。

### Raycast (System Floating Notes)
- **功能描述**：通过插件支持在桌面显示浮动信息或快速记事。
- **优点**：AI 集成度极高，生态位强大。
- **缺点**：信息展示是“瞬时”的，难以维持多个独立、常驻的数据窗口。
- **参考点**：其命令式生成（输入命令直接出结果）可作为我们 P1 阶段“自然语言生成便签”的参考。

### WidgetWall / Desktop Widgets
- **功能描述**：将网页或特定小工具（天气、股价）变成桌面组件。
- **优点**：支持将 HTML/本地 App 转化为桌面小部件。
- **缺点**：缺乏 AI 逻辑层，只是简单的“搬运”网页，无数据联动。
- **参考点**：其网页容器化技术，可借鉴用于我们扩展第三方非标准数据源。

## 差异化机会
- **机会 1：跨域数据整合**。目前没有一个产品能同时在一个视觉维度（便签）内整合“我的持仓跌幅”和“回家路上的拥堵情况”。
- **机会 2：阈值驱动的视觉反馈**。市面便签多为静态。我们将“数据变动”转化为“视觉变色”（如：ETH 跌 5% 变红，交通拥堵变红），提供生理级的直接反应。
- **机会 3：碎片信息的 AI 结构化**。目前便签只是存，不理。我们的 AI “一键整理”能将杂乱语录瞬间转化为结构化待办，具备生产力属性。

## 对 PRD 的补充建议
1. **增加“点击穿透”开关**：针对极客用户，长期挂在桌面上的便签应支持“锁定”模式，鼠标点击可直接穿透到下层窗口，避免误触。
2. **强化“数据快照”逻辑**：AI 整理不仅仅是润色，建议在整理时自动抓取该便签当前关联的实时数据（如：整理会议记录时，自动附带当时的天气或币价作为背景参数）。
3. **增加 API 健康度动效**：由于采用“本地直连”，API Key 失效或网络波动是高频痛点，建议在 Footer 增加一个微小的“呼吸灯”展示连接状态。
4. **预置“脱敏模式”**：考虑到用户可能在公共场所办公，应支持一键隐藏所有便签的具体数值（仅显示涨跌颜色或模糊化），保护个人资产隐私。
