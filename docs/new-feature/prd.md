# Market 多币种看板 + 币种详情 K 线 - PRD（定稿）

## 需求背景
用户在桌面场景下需要同时监控多个币种的价格波动（Watchlist），并在必要时快速查看特定币种的走势细节（K 线）。现有的单币种便签无法满足资产组合监控需求，而专业的交易终端又过于沉重。

## 目标用户画像
- **多持仓投资者**：需要同时盯着 BTC、ETH 及若干山寨币的实时价格。
- **轻量级交易者**：不进行高频操作，但需要通过 5m/15m K 线判断入场/离场时机。
- **办公族极客**：在工作间隙扫一眼桌面便签，获取全局行情。

## 功能清单

### P0 - 必须有
| 功能 | 描述 | 验收标准 |
| :--- | :--- | :--- |
| **多币种列表 (Watchlist)** | 单张便签支持展示多个币种的实时价格与涨跌幅。 | S/M/L/XL 分别支持展示 2/4/6/10 行数据。 |
| **严格尺寸门槛** | 限制 K 线详情页仅在 L/XL 尺寸下开启。 | S/M 尺寸点击币种行必须弹出“切换尺寸”提示层。 |
| **K 线详情页** | 展示 OHLC 蜡烛图，支持基础周期切换。 | 默认 5m 周期，支持 15m/1h 切换，数据加载不白屏。 |
| **请求合并调度** | 列表页多币种请求必须合并为单次批量请求。 | 抓包确认 10 个币种仅产生 1 次 Ticker 批量请求。 |
| **失败态快照** | 断网或限流时，列表与 K 线必须显示上次成功数据。 | 价格变灰并标注“数据更新于 X 分钟前”，趋势线置灰。 |

### P1 - 应该有
| 功能 | 描述 | 验收标准 |
| :--- | :--- | :--- |
| **基础均线 (MA)** | K 线图叠加 MA(5/10/20) 均线。 | 开关切换即时生效，不产生额外请求。 |
| **列表排序** | 支持按涨跌幅、市值或自定义顺序排序。 | 点击表头或菜单可即时重排列表。 |

### P2 - 锦上添花
| 功能 | 描述 | 验收标准 |
| :--- | :--- | :--- |
| **行内置顶/删除** | 长按或右键币种行进行快速管理。 | 交互流畅，不误触进入详情页。 |

## 用户故事（按角色分组）

### 作为普通用户...
- 我希望在桌面上挂一个小窗（S 码），一眼看到我最关心的 2 个币种价格，不占用办公空间。
- 当我发现某个币暴涨时，我希望能一键切换到大窗查看 5 分钟线，确认是否需要止盈。

### 作为运营人员...
- 我需要监控主流币种的整体情绪，列表页的红绿对比能让我快速感知市场热度。

## MVP 范围
- **只做**：多币种列表（纯数值）、K 线详情页（5m/15m/1h）、批量请求合并、尺寸切换引导。
- **不做**：列表页行内趋势图（Sparkline）、RSI/MACD 等复杂指标、1m 极高频周期、画线工具。

## 验收标准
1. **性能**：开启 10 个币种监控，内存占用增量 ≤ 20MB，CPU 占用无持续飙高。
2. **交互**：S/M 尺寸点击币种行 100% 弹出引导层；L/XL 尺寸点击 100% 进入 K 线。
3. **数据**：退出详情页后，K 线请求必须立即停止，不得在后台静默拉取 OHLC 数据。

## 简化级联分析
- **如果** 我们坚持“S/M 尺寸不画图”且“详情页按需加载”，**那么** 我们就不需要为列表页设计复杂的 Canvas 渲染引擎，也不需要维护庞大的全局 OHLC 缓存，从而大幅降低了内存占用和 API 限流风险。
- **如果** 我们统一使用“5m”作为默认周期并限制指标数量，**那么** 我们可以抽象出一套极简的图表组件，无需引入重量级的第三方交易图表库。

## PM 辩论记录
| 议题 | PM-Alpha 观点 | 老驴观点 | 最终决策 | 理由 |
| :--- | :--- | :--- | :--- | :--- |
| **列表页趋势图** | S/M 也要画 Mini Trend，增加视觉活力。 | 坚决不画。小尺寸画图是视觉灾难，且增加 CPU 负担。 | **不画** | 务实。列表页核心是数值监控，保持克制。 |
| **K 线默认周期** | 默认 1m，给用户最实时的反馈。 | 默认 5m。1m 噪音太大，且请求压力翻倍。 | **5m** | 稳健。5m 兼顾实时性与趋势判断，降低限流风险。 |
| **详情页准入** | 全尺寸支持 K 线，自动缩放。 | 仅 L/XL 支持。小窗看 K 线没有实战价值。 | **L/XL 准入** | 体验。保证 K 线图的可读性和控件操作空间。 |

## 保留的分歧
## 分歧点：指标丰富度
选项 A：仅保留 MA 均线 — 优化：性能与简洁 — 适用场景：普通盯盘用户
选项 B：增加 RSI/MACD 子图 — 优化：专业深度 — 适用场景：硬核交易用户
保留策略：v1.0 仅实现选项 A。
触发解决条件：根据 v1.0 上线后的用户反馈数据，若 30% 以上用户要求指标，则在 v1.1 引入选项 B。

[便签] 核心口径：列表页纯数值，详情页 5m K 线起步，S/M 尺寸严禁画图。
