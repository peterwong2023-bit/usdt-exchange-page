# 每日签到积分与优惠券兑换 - PRD 草案（PM-Alpha）

## 需求背景
现状（推断）：
- 用户在「USDT 兑换页」的访问具有明显的“看一眼就走”特征，缺乏低门槛的日常回访动机。
- 现有激励（若有）多集中在交易行为本身，对“轻量互动”和“习惯养成”覆盖不足。

要解决的问题：
1. 提升日活与回访：给用户一个每天回来一次的理由。
2. 促进留存与交易转化：通过积分→优惠券路径，把“轻行为”导向“重行为（兑换/交易）”。
3. 建立可控的激励体系：规则可配置、可风控、可审计，避免被薅羊毛。

需求目标：
- 用户每日签到获得积分；连续签到获得额外奖励；
- 积分可兑换优惠券（用于兑换手续费减免/汇率加成券/服务费券等，具体券种由业务定义）。

成功指标（建议）：
- 签到渗透率（DAU 中签到人数占比）
- 连签率（≥3天、≥7天、≥30天）
- 积分兑换率、券核销率
- 券带来的交易转化提升（核销用户 vs 非核销用户）
- 风控指标：异常签到占比、异常兑换占比

## 目标用户
- 高频看行情但交易不稳定的用户：需要“轻激励”维持回访。
- 新用户（注册后7天）：需要新手阶段的正反馈与路径指引。
- 价格敏感型用户：对优惠券/手续费减免强敏感。
- 沉默用户召回：用“连签断了可补救”降低回归门槛。

使用场景：
- 每日首次打开兑换页/APP/小程序时弹出签到入口
- 交易前看到“今天签到可领积分，积分可换手续费券”
- 连续签到第7天、第30天触发额外奖励与“里程碑”

## 方案对比

### 方案 A：标准签到 + 固定连签奖励 + 积分商城兑换（行业基准）
- 核心功能点
  - 签到：每日一次，领取固定积分（如 10 分）
  - 连续签到：第N天额外加成（如第7天+50，第30天+300）
  - 断签规则：断签清零或降级（常见做法：断签从1重新计）
  - 积分明细：积分获取/消耗流水
  - 积分兑换：积分商城兑换优惠券（固定面额/固定库存）
  - 优惠券：发放、到账、有效期、使用门槛、核销
  - 后台配置：每日积分值、连签奖励梯度、券库存、有效期
- 用户故事
  - 作为用户，我每天点一次签到按钮，立刻拿到积分。
  - 作为用户，我连续签到7天，拿到额外奖励，并用积分换一张手续费券。
- 创新亮点
  - 低（成熟稳定，但同质化严重）
- 潜在风险
  - 容易“签到为签到”，对交易转化拉动有限
  - 断签惩罚强导致用户挫败，连签中断后直接放弃
  - 羊毛党自动化脚本签到、批量号套利券
  - 若券价值设置不当，造成补贴不可控

### 方案 B：可配置“连签保底 + 补签机制 + 任务融合”的增长型签到（推荐）
嗬！这个想法绝了！我们能不能比竞品做得完全不一样？——把签到从“每天点一下”升级成“可控的增长发动机”：签到只是入口，积分增长与交易/关键行为轻绑定，但又不强迫。

- 核心功能点
  1. 签到基础分 + 连签加成（但加入“保底不归零”）
     - 每日基础积分（如 8~12 可配置）
     - 连签加成：线性/阶梯（第3/7/15/30天里程碑）
     - 断签不清零：改为“连签保护期/降级”  
       - 例如断签1天：连签从N降到N-2（而不是归1），降低挫败感
  2. 补签机制（用“补签卡/积分/一次性任务”换补签）
     - 用户可用少量积分兑换补签机会，或完成一次轻任务（如完成一次实名认证/绑定邮箱/完成一次兑换）获得补签卡
     - 限制：每月最多补签X次；补签不可跨月；补签需要在断签后48小时内
  3. 签到与“增长任务”融合（让积分更有方向）
     - 签到页同时展示“今日加成任务”：  
       - 例：完成一次兑换 +20；首次设置常用收款地址 +15；完成KYC +50（合规允许前提）
     - 任务可配置开关：不同地区/不同用户分层展示
  4. 积分兑换优惠券（带“智能推荐”）
     - 商城兑换：多档券（手续费券/汇率加成券/免费提现券等）
     - 推荐策略：根据用户行为推荐最可能核销的券（例如最近频繁小额兑换→推荐小门槛券）
  5. 风控与反作弊（必须配套）
     - 设备指纹/IP/行为节律识别异常签到
     - 新号兑换限制：注册后X天或完成KYC后才能兑换高价值券
     - 券领取与核销链路审计：防批量套现与倒卖
  6. 数据看板与A/B实验
     - 连签规则A/B（清零 vs 降级；固定分 vs 浮动分）
     - 券档位与门槛A/B（提高核销与转化）

- 用户故事
  - 作为新用户，我每天签到拿积分；中间断了一天也不会“全白干”，我愿意继续连签。
  - 作为活跃用户，我完成一次兑换任务获得额外积分，并能兑换到更适合我的手续费券。
  - 作为回流用户，我用一次补签卡找回连签节奏，不会因为断签而放弃。

- 创新亮点
  - “断签降级不清零”显著降低挫败感，提升长期连签率
  - 补签卡把用户从“运气断签”中救回来，提升回流与留存
  - 签到+任务融合：把补贴投向更接近交易的关键行为，ROI更可控
  - 智能推荐券：提升兑换率与核销率，避免“换了不用”

- 潜在风险
  - 规则复杂：需要清晰的UI解释与可视化（否则用户看不懂）
  - 补签可能被滥用：需严格次数限制与风控
  - 任务融合涉及合规/地区差异：需后台分层与灰度
  - 推荐策略若不准，可能降低用户满意度（“我想要的券没有/太贵”）

### 方案 C：颠覆性“组队连签挖矿式加成 + 交易池分红券”（大胆尝试）
本驴发疯一下：把“连签”做成“社交协作+池子奖励”，让用户不是为自己签到，而是为“队伍”与“共同收益”打卡。

- 核心功能点
  1. 组队连签（2~5人）
     - 成队后每日签到为队伍贡献“活跃值”
     - 队伍连续达标触发“队伍宝箱”（随机掉落：积分、券碎片、加成券）
  2. “券碎片”机制（把高价值券拆解，提升持续性）
     - 每日/每周宝箱掉落碎片，集齐可合成高价值券
  3. 交易池奖励（谨慎命名与合规）
     - 队伍每周满足“队伍活跃 + 交易达标”可获得“周度加成券”
     - 注意：不要设计成“分红/收益承诺”，避免合规风险；仅作为平台补贴券
  4. 反作弊加强
     - 队伍成员同设备/同IP聚集风险更高，需要更严格的限制
     - 队伍奖励与个人奖励拆分，避免刷队套利

- 用户故事
  - 作为用户，我拉上朋友组队签到，为了开宝箱坚持每天回来。
  - 作为队长，我想维持队伍连签，提醒队友签到以获得周度奖励券。
- 创新亮点
  - 社交绑定显著提升留存（“我不签到会拖累队友”）
  - 碎片化奖励提高长期目标感，减少一次性薅完就走
- 潜在风险
  - 合规与运营风险：容易被误解为“拉人头/收益承诺”
  - 复杂度高：用户学习成本、客服成本上升
  - 羊毛党组队刷奖励更隐蔽，需要更强风控与限制

## 推荐方案及理由
推荐 **方案 B：增长型签到（连签保底 + 补签 + 任务融合 + 智能券推荐）**。

理由：
1. **效果更稳**：相比A的同质化，B能显著提升连签与回访（断签不清零、可补签），而不必依赖强社交裂变。
2. **ROI更可控**：通过“任务融合”把更多积分投向更接近交易转化的行为，避免纯签到空转。
3. **风险更可管**：相比C的社交挖矿式玩法，B在合规、风控、用户理解成本上更可控，适合先落地验证。
4. **可渐进演进**：B上线后，如果数据证明社交能有效提升留存，再从B平滑扩展到轻量组队/分享，而不是一步到C的高风险复杂系统。

## 用户故事（推荐方案详细版）
1. 每日签到（基础）
   - 作为用户，我进入兑换页首页，看到“今日签到领积分”入口；
   - 点击签到后，立即看到本次获得积分、当前积分余额、连签天数与下一个里程碑奖励。

2. 连签里程碑（激励）
   - 作为用户，我连续签到第7天，系统弹出“第7天里程碑奖励”，并展示“第15天可得XX”的明确预期。

3. 断签降级（降低挫败）
   - 作为用户，我不小心断签一天，系统提示“连签从12天降为10天（保护机制）”，并说明原因与规则；
   - 我仍然愿意继续签到，而不是放弃。

4. 补签（召回）
   - 作为用户，我断签后48小时内进入签到页，看到“补签找回连签”的选项；
   - 我可以使用一张补签卡（或消耗X积分）完成补签，连签恢复到断签前水平（或按规则恢复）。

5. 任务加成（导向转化）
   - 作为用户，我在签到页看到“完成一次兑换 +20积分”的今日任务；
   - 我完成兑换后，积分自动入账，并提示我可兑换某张手续费券。

6. 积分兑换与用券闭环
   - 作为用户，我进入积分商城，看到推荐的优惠券（例如“手续费-xx%券”）；
   - 我用积分兑换成功后，优惠券进入“我的卡券”，并在下单/兑换时自动可选或自动最优匹配（可配置）。

## 初步验收标准
1. 签到与积分发放
   - 每个用户每天仅可签到一次（按配置时区/自然日定义），重复请求不重复发放
   - 签到成功后：积分余额、积分流水、连签天数实时更新（允许秒级延迟）

2. 连签计算与断签规则
   - 连签按自然日累计；断签触发“降级”而非清零（降级规则可配置并可回归测试覆盖）
   - 里程碑奖励在达成当日仅发放一次，补签触发里程碑时同样正确发放（或按配置禁止补签触发里程碑，需明确）

3. 补签机制
   - 补签入口仅在满足条件时出现：断签后<=48小时、当月补签次数未超限
   - 补签消耗（补签卡或积分）扣减正确，且有完整流水
   - 补签不可跨月；跨月时连签策略按规则重置

4. 积分商城与优惠券
   - 积分兑换：库存扣减正确、并发安全（防超卖）、失败回滚
   - 优惠券：到账可见、有效期正确、使用门槛正确、核销后状态正确
   - 下单/兑换时：优惠券可选/自动匹配逻辑符合配置（至少保证“可用券展示正确”）

5. 风控与限制
   - 新用户高价值券兑换限制生效（如注册天数/KYC/交易次数门槛）
   - 异常签到/兑换触发风控策略：限频、验证码、人工审核或封禁（至少具备可扩展策略位）
   - 管理后台具备：规则配置、券配置、库存、用户积分查询与流水审计

6. 数据埋点（最小集合）
   - 曝光：签到入口曝光、签到页曝光、任务曝光、商城曝光
   - 行为：签到点击与成功、补签点击与成功、兑换点击与成功、券核销
   - 关键漏斗可还原：曝光→签到→积累→兑换→核销→交易转化
