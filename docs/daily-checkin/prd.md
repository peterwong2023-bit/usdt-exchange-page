# 每日签到积分与优惠券兑换 - PRD（定稿）

## 需求背景
当前USDT兑换页用户访问模式以"看一眼就走"为主，缺乏日常回访动机。需要建立低门槛的用户留存机制，通过每日签到积分系统提升用户粘性，并通过积分兑换优惠券促进交易转化。

关键指标：提升日活回访率、增强用户留存、促进交易转化。

## 目标用户画像
- **价格敏感型用户**：对手续费减免、汇率优惠敏感
- **高频浏览用户**：经常查看行情但交易不稳定
- **新用户**：需要正反馈建立使用习惯
- **流失召回用户**：需要低门槛重新激活

## 功能清单

### P0 - 必须有（MVP范围）
| 功能 | 描述 | 验收标准 |
|------|------|----------|
| 每日签到 | 用户每天可签到一次获得积分 | 每用户每日仅可签到一次，积分正确发放 |
| 连续签到计数 | 记录用户连续签到天数 | 连签天数正确计算，断签后重置为1 |
| 基础连签奖励 | 连续3/7/15/30天给予额外积分 | 里程碑奖励正确发放且不重复 |
| 积分余额管理 | 显示用户当前积分及流水 | 积分增减记录完整，余额实时更新 |
| 积分商城 | 积分兑换固定面额优惠券 | 库存扣减正确，防超卖，兑换记录完整 |
| 优惠券管理 | 券发放、有效期、使用状态管理 | 券状态正确，过期自动失效 |
| 基础风控 | IP/设备限制，新用户兑换限制 | 异常行为识别，高价值券兑换门槛 |

### P1 - 应该有（v1.1版本）
| 功能 | 描述 | 验收标准 |
|------|------|----------|
| 断签保护 | 断签后连签天数降级而非清零 | 断签1天连签-2天，降低挫败感 |
| 补签机制 | 断签后48小时内可补签 | 限制每月补签次数，消耗积分或补签卡 |
| 券使用提醒 | 交易时提醒可用券 | 下单页面展示可用优惠券 |
| 管理后台 | 规则配置、用户查询、数据统计 | 支持积分规则调整、用户数据查询 |

### P2 - 锦上添花（数据验证后考虑）
| 功能 | 描述 | 验收标准 |
|------|------|----------|
| 任务加成 | 完成特定行为获得额外积分 | 与其他系统联动，需评估技术成本 |
| 智能券推荐 | 根据用户行为推荐券种 | 需要足够数据支撑推荐算法 |
| 签到日历 | 可视化签到记录 | 提升用户体验，非必需功能 |

## 用户故事（按角色分组）

### 作为普通用户
- 我希望每天签到获得积分，操作简单明确
- 我希望连续签到有额外奖励，但断签后不会完全白费
- 我希望积分能兑换到实用的优惠券
- 我希望在交易时能方便使用优惠券

### 作为新用户
- 我希望快速理解签到规则，不需要复杂学习
- 我希望前期奖励相对丰富，建立正反馈
- 我希望兑换门槛不会太高，能快速体验到好处

### 作为运营人员
- 我需要能灵活调整签到积分数量
- 我需要监控券的发放和使用情况
- 我需要识别和处理异常签到行为
- 我需要分析功能效果数据

### 作为管理员
- 我需要完整的用户积分流水记录
- 我需要风控规则配置能力
- 我需要券库存管理和补充功能

## MVP 范围
**v1.0 只做：**
- 标准每日签到积分发放
- 简单连签奖励（3/7/15/30天）
- 断签清零重置（降级机制v1.1再加）
- 基础积分商城（2-3种券）
- 核心风控（IP/设备/新用户限制）

**v1.0 不做：**
- 复杂任务系统
- 社交组队功能
- 智能推荐算法
- 补签机制
- 高级风控策略

## 验收标准

### 功能验收
1. 签到功能稳定性：支持高并发，响应时间<2秒
2. 积分计算准确性：所有积分增减操作有完整审计日志
3. 券系统完整性：兑换-发放-使用-核销闭环无误
4. 风控有效性：能识别基础异常行为并拦截

### 数据埋点
- 签到相关：签到页曝光、签到成功、连签达成
- 兑换相关：商城曝光、券兑换成功、券使用成功
- 转化漏斗：签到用户→积分积累→券兑换→券使用→交易转化

### 运营指标
- 签到渗透率：DAU中签到用户占比 >30%
- 连签率：连续7天签到用户占比 >15%
- 券兑换率：积分用户中兑换券占比 >20%
- 券核销率：已兑换券的使用占比 >60%

## PM 辩论记录

| 议题 | PM-Alpha 观点 | PM-Beta 观点 | 最终决策 | 理由 |
|------|---------------|-------------|----------|------|
| 断签处理 | 降级不清零，降低挫败感 | MVP先用清零，降级v1.1加 | 采用PM-Beta | 降级机制增加复杂度，先验证基础功能效果 |
| 任务融合 | 签到+任务提升ROI | 任务依赖其他系统，风险高 | 采用PM-Beta | v1.0聚焦核心签到，任务系统后续评估 |
| 社交组队 | 社交提升留存效果好 | 合规风险高，复杂度激增 | 采用PM-Beta | 风险收益比不合理，砍掉 |
| 智能推荐 | 提升兑换率和满意度 | 算法复杂，数据不足 | 采用PM-Beta | 先做固定券种，积累数据后再考虑推荐 |
| 风控策略 | 需要完善反作弊 | 基础风控即可，过度风控影响体验 | 折中方案 | IP+设备+新用户限制，逐步完善 |
| 券种选择 | 多样化券种满足不同需求 | 2-3种核心券种即可 | 采用PM-Beta | MVP阶段券种不宜过多，运营复杂度可控 |
