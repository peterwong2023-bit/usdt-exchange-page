# 积分兑换商品 - PRD（定稿）

## 需求背景
目前用户积分处于“获取易、消耗难”的阶段，缺乏实质性的兑换出口导致积分价值感低，无法有效带动存量用户的活跃与复购。本功能旨在建立一套闭环的积分消耗体系，通过实物与虚拟商品的兑换，提升用户粘性。

老驴我审核了 Alpha 的提案，方案 B 整体务实，但“智能推荐”和“候补机制”在 v1.0 投入产出比太低，先砍掉。稳扎稳打，先把核心兑换链路和能闭环拉动的“差额补齐”做实。

## 目标用户画像
- **羊毛党用户**：积分较少，盯着低门槛商品（签到礼包、小额券）。
- **核心活跃用户**：积攒了大量积分，追求高价值或独家定制实物。
- **运营人员**：需要灵活配置商品、控制成本、处理售后。

## 功能清单

### P0 - 必须有
| 功能 | 描述 | 验收标准 |
| :--- | :--- | :--- |
| 商品管理系统 | 支持实物/虚拟商品上架、分类、图文详情、库存设置 | 商品状态（上架/下架）实时生效，库存不准超卖 |
| 基础兑换流程 | 浏览-详情-扣分下单-收货地址（实物）-结果反馈 | 扣分与订单生成必须原子化，失败需自动回滚积分 |
| 积分流水对账 | 记录每一笔兑换产生的积分扣减、退回详情 | 流水包含订单号、操作时间、变动前/后分值 |
| 基础风控 | 限制单人单品兑换次数、单日兑换总限额 | 触发限购时前端明确提示，后台可配置黑名单 |
| 订单管理 | 后台支持发货操作、物流单号录入、手动取消订单 | 虚拟商品兑换后需提供卡密/权益查看入口 |

### P1 - 应该有
| 功能 | 描述 | 验收标准 |
| :--- | :--- | :--- |
| 差额补齐（混合支付） | 积分不足时，支持“积分+现金”组合支付 | 支持现金退款原路返回，积分退回原账户 |
| 差额引导任务 | 详情页提示“还差X分”，点击跳转至积分任务中心 | 任务页需有回跳机制，确保用户能快速返回兑换 |
| 兑换专题页 | 支持 Banner、金刚位配置，展示爆款或节日专题 | 运营后台可自定义页面布局和商品排序 |

### P2 - 锦上添花
| 功能 | 描述 | 验收标准 |
| :--- | :--- | :--- |
| 自动取消订单 | 未完成支付或超出的待处理状态订单定时释放 | 超时时长可配置（默认30分钟） |
| 智能价值锚点 | 在商品页自动计算并展示“1积分≈0.01元”等价值提示 | 仅针对特定对比商品展示，提高兑换动机 |
| 候补通知 | 商品售罄后用户可订阅补货通知 | 补货后通过站内信或 Push 触达用户 |

## 用户故事

### 作为普通用户...
- 我希望在积分不足时能通过支付少量现金或完成简单的签到任务补齐，这样我就能立刻换走心仪的商品，不用等遥遥无期的攒分过程。
- 我希望兑换虚拟奖品（如视频会员、优惠券）后能立即看到卡密，不需要等待人工审核。

### 作为运营人员...
- 我需要针对不同等级的用户展示不同的商品池，避免高价值礼品被新号快速褥走。
- 我需要实时监控商品的兑换率和运营成本，一旦补贴超出预算，能一键通过后台下架相关商品。

### 作为商户/供应链人员...
- 我需要能批量导出待发货订单，导出格式需符合物流打单软件要求，减少人工录入环节。

## MVP 范围
**v1.0 核心聚焦：**
1. 全链路兑换：实物（含地址）+ 虚拟（含卡密）的闭环。
2. 基础风控：防止脚本刷单和超卖。
3. 后台协同：满足最基本的发货、退分、配置需求。
**不做：** 智能推荐、积分拍卖、组团兑换（ROI 不明，逻辑复杂）。

## 验收标准
1. **并发性**：在双 11 等秒杀场景下，支持至少 1000TPS 的兑换请求，严禁出现负库存。
2. **数据一致性**：若调用支付接口失败或网络超时，扣除的积分必须在 5 分钟内通过对账补偿机制返还。
3. **兼容性**：移动端 H5 页面在主流 Android 和 iOS 浏览器下加载时间不得超过 1.5s。

## PM 辩论记录
| 议题 | PM-Alpha 观点 | 老驴观点 | 最终决策 | 理由 |
| :--- | :--- | :--- | :--- | :--- |
| 智能推荐 | 提高点击和转化，提升用户体验 | 初始商品量少，搞算法是杀鸡用牛刀，增加研发成本 | **砍掉** | 先用运营手动排序人工干预，等 SKU 过百再议。 |
| 候补/预约 | 降低缺货流失，增加回流 | 规则复杂，涉及分布式锁和通知系统，v1.0 开发周期长 | **放入 P2** | 优先保证“能换到”，缺货说明由于运营预估不足，应从源头解决。 |
| 积分拍卖 | 增加互动性，消耗大笔积分 | 法律合规风险高（类似赌博），用户认知门槛高 | **砍掉** | 稳字当头，不要挑战监管底线。 |
| 混合支付 | 提高转化，增加现金流 | 涉及支付合规和财务对账，逻辑复杂 | **保留 P1** | 这是提升兑换率的核武器，ROI 高，值得投入研发。 |
