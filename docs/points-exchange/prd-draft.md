# 积分兑换商品功能 - PRD 草案（PM-Alpha）

## 需求背景
用户在完成签到、消费、任务、互动等行为后积累积分，但常见痛点是：
- **积分“看不见价值”**：不知道能换什么、差多少、值不值，导致积分沉睡。
- **兑换链路长/不确定**：库存不足、发货慢、规则不透明，引发投诉。
- **运营效率低**：商品上新、定价、库存、活动配置复杂，ROI 不清晰。
- **增长拉动弱**：积分仅做“福利”，无法反哺拉新、复购、留存。

目标：搭建一套可运营、可增长、可控风险的「积分兑换商品」能力，让积分成为可感知的“第二货币”，同时沉淀可复用的营销基建。

## 目标用户
- **C 端用户**
  - 积分新手：积分少、想快速体验“能换到东西”
  - 积分重度用户：积分多、追求高价值/稀缺商品
  - 价格敏感用户：愿意用积分抵扣，追求“占便宜确定性”
- **B 端运营/商品/客服**
  - 运营：配置活动、爆品、提升兑换率与留存
  - 商品：管理商品、库存、供应链、成本
  - 客服：处理失败兑换、缺货、发货与售后

## 方案对比

### 方案 A：标准积分商城（基准款）
- 核心功能点
  - 积分商城首页：分类/搜索/排序（热门、所需积分、上新）
  - 商品详情：所需积分、库存、兑换规则、发货说明
  - 兑换下单：确认收货地址（实物）/发放码（虚拟）/权益即时到账
  - 订单列表：待发货/已发货/已完成/已取消
  - 库存与风控：限购、黑名单、反作弊（同设备/同地址/频次）
  - 运营后台：商品配置、上下架、库存、积分定价、banner、公告
- 用户故事
  - 用户看到可兑换商品 → 用积分下单 → 填地址 → 等待发货 → 完成
- 创新亮点
  - 无（行业通用、交付确定性强）
- 潜在风险
  - 兑换率提升有限：用户仍觉得“不值/不够”
  - 库存与供应链压力集中在运营侧
  - 容易陷入“比谁商品多”的同质化竞争

### 方案 B：智能兑换 + 成长引导（强运营增长款）
嗬！这个想法绝了：不只是“能换”，而是让用户**更快换到、换得更值、差一点也能换**，把积分从静态货币变成“可被引导的成长路径”。

- 核心功能点
  - **差额补齐（混合支付）**：积分不足时可用少量现金/优惠券补齐（可配置是否开启）
  - **“差 X 分可兑换”提醒**：商品卡片展示距兑换差额；一键去做任务补齐
  - **个性化推荐**：基于用户积分区间、偏好品类、历史兑换推荐“最可能成交”的 10 个商品
  - **智能兑换包**：用固定积分兑换“礼包”（运营可配置：小奖池/大礼包），提升低积分用户成就感
  - **预约/候补机制**：缺货商品可预约，补货自动通知；支持“积分冻结占位”（可选）
  - **价值锚点**：展示参考价值（市场价/券后价/历史兑换热度）+ 兑换划算度标签（运营可控）
  - 运营后台增强
    - 商品“成本-积分”测算：毛利/补贴/兑换率预估
    - 任务与商品联动：配置“补齐任务”（签到、浏览、分享、下单等）
    - 分层投放：按积分段/新老用户/地域投放不同商品池
- 用户故事
  - 用户看中某商品但差 120 分 → 页面提示“做 2 个任务即可兑换” → 完成任务立刻兑换
  - 用户积分很少 → 兑换“新手礼包”获得小确幸 → 形成持续攒分动力
  - 爆品缺货 → 用户预约候补 → 补货自动通知并预留 30 分钟兑换窗口
- 创新亮点
  - 把“兑换”变成“可引导的转化漏斗”：看中 → 补齐 → 成交
  - 低积分用户也能快速体验“兑换成功”，显著提升留存与任务完成率
  - 预约/冻结机制缓解爆品库存焦虑，提升确定性
- 潜在风险
  - 混合支付引入支付合规与退款边界（需明确：积分退回/现金退回规则）
  - 个性化推荐需要数据与冷启动策略
  - 冻结占位可能被薅（需反作弊与超时释放）

### 方案 C：积分“抢拍/集体兑换”市场（颠覆性玩法款）
本驴的脱缰方案：让积分商城从“超市”变成“交易所/秀场”，用稀缺与社交驱动参与，做出完全不同的心智。

- 核心功能点
  - **积分拍卖场**：稀缺商品以低起拍积分进入，用户出价；到点最高价得（可设置保留价/封顶价）
  - **组团兑换（众筹式）**：商品设置“成团门槛”（如 100 人各付 300 分），成团即发货，不成团自动退分
  - **积分盲盒赛季**：每周主题盲盒（数码周/生活周），概率透明+保底机制（可监管）
  - **荣誉体系**：拍卖榜、组团发起人勋章、兑换战绩卡（可分享）
- 用户故事
  - 用户用较少积分参与拍卖 → 竞价刺激持续回访 → 竞拍成功获得“战绩卡”分享
  - 用户发起组团兑换 → 拉好友参团 → 成团后全员得货，发起人额外奖励
- 创新亮点
  - “稀缺+博弈+社交”显著提高 DAU 与停留时长
  - 组团兑换可反向验证需求，降低备货风险
- 潜在风险
  - 规则复杂、用户认知成本高
  - 拍卖/盲盒存在监管与舆情风险（需概率公示、限制人群、风控）
  - 容易偏离“兑换福利”的原始定位，引发公平性争议

## 推荐方案及理由
推荐 **方案 B：智能兑换 + 成长引导**。

理由（偏结果导向）：
1. **最稳地提升兑换转化**：差额补齐 + 补齐任务闭环，直接解决“差一点换不到”的核心流失点。
2. **兼顾用户体验与运营可控**：礼包/候补/推荐都是可配置能力，能随预算与供给动态调整。
3. **风险可控、可渐进演进**：先做标准兑换能力，再逐步打开混合支付、冻结占位、个性化，不需要一次性押注高风险玩法。
4. **为未来玩法打底**：B 的订单、库存、风控、任务联动是 C 的基础设施，后续可小流量灰度拍卖/组团。

## 用户故事（推荐方案详细版）
1. **新手快速首兑**
   - 作为新用户，我进入积分商城看到“新手 500 分礼包”，兑换成功后立即获得小礼品/优惠券，从而相信积分有用并愿意继续攒分。
2. **差额补齐促成交易**
   - 作为积分不足的用户，我在商品详情看到“差 120 分”，系统提示可“做任务补齐”或“积分+现金补齐”，我选择做任务补齐并完成兑换。
3. **缺货不流失**
   - 作为想要爆品的用户，当商品售罄时我可点击“预约候补”，补货后收到通知并在限时窗口内兑换，避免反复刷新与流失。
4. **权益类即时到账**
   - 作为兑换虚拟权益的用户，我希望兑换后立即获得兑换码/权益到账记录，并在订单中可随时查看与复制使用。
5. **运营快速上新与控补贴**
   - 作为运营人员，我可以配置商品积分价、库存、限购、投放人群与补齐任务；并看到兑换率、补贴成本、缺货率，从而调整策略。
6. **异常与售后可追溯**
   - 作为客服人员，我可以查询用户兑换订单、发货状态、冻结/退回记录，必要时发起退分/补发，降低扯皮成本。

## 初步验收标准
1. **兑换链路**
   - 用户可完成：浏览列表 → 查看详情 → 确认兑换 → 生成订单 → 状态流转（待发货/已发货/已完成/已取消）。
   - 实物订单必须支持地址管理；虚拟商品必须支持“码/权益”展示与二次查看。
2. **积分扣减与一致性**
   - 下单成功即扣减积分；取消/失败按规则退回积分（含冻结场景则释放）。
   - 积分流水可追溯：时间、类型（扣减/退回/冻结/释放）、关联订单号。
3. **库存与限购**
   - 库存扣减与订单创建原子性：不允许超卖。
   - 支持配置：每人限兑 N 次/每 SKU 限兑/每自然日限兑；触发时前端明确提示。
4. **差额补齐（可开关）**
   - 商品可配置是否支持混合支付；开启后展示补齐入口与费用明细。
   - 混合支付订单取消/退款规则清晰：积分与现金分别按原路退回（或按配置策略），订单页可见。
5. **补齐任务联动**
   - 在“差 X 分”场景下，支持一键跳转任务页；完成任务后积分实时更新并回跳可继续兑换。
6. **预约候补**
   - 售罄商品可预约；补货后触达（站内信/Push/短信按配置）；
   - 预约用户获得限时兑换窗口（如 30 分钟，可配置），超时自动释放资格。
7. **运营后台**
   - 支持商品新增/编辑/上下架、积分定价、库存、限购、投放人群、banner 配置；
   - 提供基础报表：曝光、点击、兑换转化率、缺货率、订单履约时效、补贴成本（按商品维度）。
8. **风控与反作弊（基础版）**
   - 同账号/同设备/同地址频次限制可配置；
   - 异常兑换命中后可拦截并记录原因（便于客服与复盘）。
9. **体验与性能**
   - 列表首屏加载 ≤ 2s（常规网络条件）；关键操作（兑换提交）接口 99 线 ≤ 800ms（以内部 SLA 为准）。
10. **灰度与回滚**
   - 支持按用户分组灰度开启：混合支付、候补冻结、个性化推荐；具备快速关闭开关。
