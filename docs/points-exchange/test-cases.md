# 积分兑换商品 - 测试用例

## 测试概要
- **测试范围**：商品管理、兑换流程（实物/虚拟）、积分对账、风控限购、混合支付、任务引导、后台订单管理。
- **前置条件**：用户已登录，账户已有初始积分，后台已配置测试商品。
- **测试环境要求**：包含网关、订单、资产、库存、风控各微服务环境；支持 Mock 支付接口；具备多终端测试工具。

| 编号 | 模块 | 用例名称 | 前置条件 | 测试步骤 | 预期结果 | 优先级 |

## 性能测试建议

| 指标 | 基准要求 | 测试方法 |
|------|----------|----------|
| 核心接口并发 | 1000 TPS | 使用 JMeter 模拟高并发下单，观测 DB 行锁死锁率 |
| 页面加载性能 | < 1.5s (H5) | 使用 Lighthouse 测量 FCP 和 LCP 指标 |
| 下单响应时间 | 95 线 < 500ms | 压测下单全链路，观察库存与积分服务响应 |

## 回归测试清单
1. **现有支付系统**：确保积分混合支付不影响纯现金交易。
2. **任务中心模块**：确保积分兑换带来的积分变动能正确触发任务状态更新。
3. **用户资产系统**：确保积分冻结与解冻逻辑不影响用户的其他积分消耗场景（如抽奖等）。
