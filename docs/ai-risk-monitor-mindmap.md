> 版本：v2.0 | 更新：2026-03-05
> 用途：产品需求核对、开发实现参考、测试用例基础
> 状态：✅ 已确认 | ⏳ 待确认 | 📌 待后续讨论

---

## 系统定位

- **核心定位**：风险优先级排序工具（非自动封号系统）
- **运行模式**：Observe Only — 仅展示，不触发任何自动处理
- **核心逻辑**：风险事件 → AI模型权重排序 → 运营看到事件+AI推荐分数 → 人工判断
- **核心场景**：盗号 / 骗号检测 ✅
- **使用约束**：
  - 单一信号 ≠ 风险
  - 单一规则 ≠ 风险用户
  - 必须"画像 + 行为 + 安全信号"组合
  - 输出为风险事件，由系统或人工处理

## A. 用户画像层（5维画像）✅

> 画像本身不产生风险，是上下文——帮运营理解"这个用户是什么样的人"

### A1. 账户属性画像 ✅

| 维度 | 定义 | 状态 |
|------|------|------|
| 注册阶段 | 新注册 ≤ 30天 / 成熟 > 30天（2档） | ✅ |
| 交易角色 | 未交易 / 买家 / 卖家 / 双角色 | ✅ |
| 高净值标签 | 当前资产 ≥ 50,000（默认，可配置） | ✅ |

### A2. 活跃度画像 ✅

| 维度 | 定义 | 状态 |
|------|------|------|
| 活跃水平（30天窗口） | 固定阈值（第一阶段），后续可切分位数 | ✅ |
| 活跃周期 | 短期(30天) / 中期(60天) / 长期(90天) | ✅ |
| 新用户处理 | 注册 ≤ 30天不参与活跃度计算，显示"暂无评级" | ✅ |
| 休眠判定 | 90天无交易 | ✅ |

**计算方案**：
- 第一阶段：固定阈值（如 ≥20笔/月=高活跃），可在风控管理台配置
- 第二阶段：每日离线计算分位数 + 缓存，阈值自动更新
- 排除 ≤ 30天注册用户，避免拉低基线

### A3. 资金流向画像 ✅

| 维度 | 定义 | 状态 |
|------|------|------|
| 有效资金门槛 | 默认值待定（平台为成熟期），可在风控管理台配置 | ✅ |
| 平台资金方向 | 净流入(充值≥提现×1.5) / 净流出(提现≥充值×1.5) / 平衡 | ✅ |
| 平台内交易方向 | 以买入为主 / 以卖出为主 / 买卖平衡 / 未交易 | ✅ |
| 资金数值展示 | 30天充值/提现/买入/卖出总额 | ✅ |
| 资金行为标签 | 高频充值(≥100次) / 高频提现(≥100次) / 小额高频充值(≥30次且≤200) | ✅ |

**设计决策**：
- 1.5倍系数保持，宽松减少噪音，真正的风险在规则层判断
- 有效资金门槛做成可配置参数

### A4. 交易习惯画像 ✅

| 维度 | 定义 | 状态 |
|------|------|------|
| 交易规模 | 大额(单笔≥50,000) / 持续大额(连续10笔≥50,000) | ✅ |
| 交易粒度 | 小额(≤200) / 小额密集(占比≥80%) | ✅ |
| 交易对象集中度 | 30天内 ≥70% 交易来自同一对手（按单一用户ID计算） | ✅ |
| 交易策略 | 囤币(买入后30天无卖出) / 快进快出(周期≤24h且≥3次) | ✅ |
| 交易节律 | 节律集中(70%集中固定时段) / 固定间隔(波动≤10%) | ✅ |

### A5. 安全画像 ✅

| 维度 | 定义 | 状态 |
|------|------|------|
| 安全事实(长期) | IP/手机号/身份证/二维码/支付方式被拉黑 / 设备多高危账号 | ✅ |
| 处置结果 | 冻结次数 / 拉黑次数 / 身份证关联账号数 | ✅ |
| 行为信号(周期) | 人脸失败≥2次/24h / 被举报≥3次/30d / 连续取消≥3次 / 卖家同意后取消 | ✅ |
| 展示方式 | **累计式**（总历史 + 近期窗口数据） | ✅ |

## B. 安全行为风险层（设备层）✅

> 权重：40%（子域封顶40分）
> 核心目标：发现"不是本人在操作"的迹象

### B1. 架构前提 ✅

| 项目 | 说明 |
|------|------|
| Fingerprint | 付费版已接入 |
| 三端采集 | App(Flutter): visitorId + confidence + os + device；WebView/Browser: visitorId + confidence |
| 服务端 | Smart Signals全量获取（VPN/Proxy/Tor/Bot等） |
| 账号体系 | **三端隔离**：同一手机号在App/Web/WebView是不同账号ID |
| 跨端关联 | 通过手机号关联，visitorId不做跨端关联 |
| 每个visitorId | 独立看待，等于一个独立身份 |

### B2. 设备指纹信号 ✅

| 信号 | 条件 | 可配置 | 备注 |
|------|------|--------|------|
| NEW_DEVICE | 首次出现该visitorId | — | ⏳ **待确认：相对账号还是全平台** |
| LOW_CONFIDENCE | confidence < 0.6 | ✅ | 检测反指纹工具/模拟器 |
| RAPID_DEVICE_SWITCH | 24h内 ≥ 2个visitorId | ✅ | 天然单端，无跨端干扰 |
| SHARED_DEVICE | 7天内 ≥ 3个账号使用同一visitorId | ✅ | 群控/工作室检测 |
| HEAVY_SHARED_DEVICE | 30天内 ≥ 5个账号使用同一visitorId | ✅ | 重度共享设备 |

### B3. IP/地理环境信号 ✅

| 信号 | 条件 | 数据源 | 可配置 |
|------|------|--------|--------|
| CITY_JUMP | 当前城市≠上次城市，距离≥300km | 自有IP解析 | ✅ |
| FREQUENT_CITY_CHANGE | 24h内切换≥3个城市 | 自有IP解析 | ✅ |
| IP_MULTI_ACCOUNT | 24h内同一IP≥3个账号 | 自有IP解析 | ✅ |
| IP_ABNORMAL | 数据中心IP/代理/异常ASN | 自有 + Fingerprint | — |
| ANON_NETWORK | VPN/Proxy/Tor = true | Fingerprint Smart Signals | — |

**IP双数据源搭配**：
- 自有IP解析：实时、全量，用于CITY_JUMP / IP_MULTI_ACCOUNT基础判定
- Fingerprint Smart Signals：增强，VPN=true时自有IP城市不可信
- CITY_JUMP + VPN=true → 风险更高（故意隐藏位置）

### B4. 行为时机信号 ✅

| 信号 | 条件 | 备注 |
|------|------|------|
| LOGIN_BURST | 10分钟内登录/验证失败≥3次 | 暴力破解检测 |
| SENSITIVE_ACTION_AFTER_LOGIN | 新设备登录后≤10分钟内执行敏感操作 | 盗号标志性路径 |

**敏感操作列表（5项）** ✅：
1. 提现
2. 卖出（特别是全部余额）
3. 修改登录密码
4. 修改支付密码
5. 绑定/修改收款方式

## C. 交易行为安全层 （部分确认）

> 权重：35%（子域封顶35分）

### C1. 交易速度信号 ✅

| 信号 | 条件 | 说明 |
|------|------|------|
| LOGIN_FAST_SELL | 登录后≤10分钟内发起多笔卖单 | 与SENSITIVE_ACTION分开保持，独立信号 |
| LOGIN_CLEAR_BALANCE | 登录后立即卖出全部可用余额 | 清空账户 |
| ORDER_PAY_TOO_FAST | 卖家点同意 → 买家点已付款 ≤ 3秒 | 盗号自导自演，极强信号 |

### C2. 取消与超时信号 （部分确认）

| 信号 | 条件 | 说明 | 状态 |
|------|------|------|------|
| ORDER_CANCEL_BURST_5M | 5分钟内取消≥3次 | 干扰卖家 | 📌 待讨论 |
| ORDER_CANCEL_BURST_30M | 30分钟内取消≥5次 | 干扰卖家 | 📌 待讨论 |
| ORDER_CANCEL_AFTER_ACCEPT | 卖家同意后买家取消≥1次 | **套取卖家支付信息**（强信号） | ✅ |
| ORDER_CANCEL_LOOP | 下单→取消≥3轮 | 循环骚扰 | 📌 待讨论 |
| ORDER_PAY_NEAR_TIMEOUT | 近30天≥70%订单接近超时才付款 | — | 📌 待讨论 |
| SELLER_CANCEL_FLOOD（新增） | 同一卖家30分钟内被≥3个不同买家取消 | **卖家视角：正在被攻击** | ✅ |

**已确认的骗术场景**：
- 卖家同意后取消 → 骗子截取卖家收款二维码/银行账号，场外行骗
- 高频取消 → 干扰卖家制造混乱，用其他号趁乱行骗
- 收款信息在卖家同意后才展示 ✅

### C3. 交易关系信号 📌 待讨论

## E. AI风控评分体系 ✅

### E1. 评分模型

| 项目 | 说明 |
|------|------|
| 输入 | 风险事件（安全/交易/团伙/画像4类） |
| 单事件得分 | event_score = base_score × confidence × decay_factor |
| 评分等级 | ≥80 高危 / 60-80 中危 / <60 低危（3级） |
| 可解释性 | 必须输出Top风险因子（事件名/贡献分/时间/所属域） |

### E2. 子域评分（加权 + 封顶）

| 子域 | 权重 | 封顶 |
|------|------|------|
| 安全行为 | 40% | ≤ 40分 |
| 交易行为 | 35% | ≤ 35分 |
| 团伙检测 | 20% | ≤ 20分 |
| 用户画像 | 5% | ≤ 5分 |

### E3. 展示方式 ✅

- 子域进度条（安全 38/40 + 交易 33/35 + 团伙 18/20 + 画像 2/5 = 91）
- Top风险因子列表（贡献分 + 所属域 + 简要描述）
- 规则层维持独立按域展示（安全层/交易层/团伙层）

### E4. 反馈闭环 ✅

- 人工审核结果回流 → 模型训练
- 稽查通过时提示"判定结果将回流至AI模型训练"
- AI模型卡片显示"本月审核回流 + 正/负样本比"

### F2. 组合规则列表

| 规则 | 条件 | 输出 |

| R-ATO-3 | NEW_DEVICE + SENSITIVE_ACTION | 新设备+敏感操作 |

|------|------|
| 顶部KPI | **工单效率导向**：待接单 / 受理中 / 待稽查 / 今日已完结 |
| 筛选器（两行） | 第一行：优先级/风险等级/风险类型/搜索/查询/重置 |
| | 第二行：时间范围（日期选择器+快捷：今天/本周/本月/本季/本年） |

**工单流转** ✅：
```
待接单 →[接单]→ 受理中 →[审核判定]→ 待稽查 →[稽查]→ 已完结
                  ↓                               ↓
               [解单]→ 待接单                 [驳回]→ 待接单
```

**工单操作**：
| 操作 | 交互 | 说明 |
|------|------|------|
| 接单 | 直接生效（无确认） | 锁定给当前用户 |
| 解单 | 弹窗确认（无需原因） | 释放回待接单池 |
| 审核 | 打开详情页 | 进行判定 |
| 稽查 | 打开详情页（稽查模式） | 通过/驳回 |
