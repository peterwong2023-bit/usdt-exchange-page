# 2025 信用分计算修订版 v1.1

> 基于 2024 版信用体系的渐进式升级方案（v1.1 深度修订版）
> 设计目标：完善用户信用体系 + 提升交易匹配精准度 + 优化数值平衡与反作弊落地
> 用户感知：完全静默迁移，老用户体验零变化，界面仅展示平滑分。

---

## 0. 设计原则

1. **静默迁移与平滑展示**：用户看到的等级、信用分、升级条件保持直觉稳定（双轨制显示）；规则只在后台改。
2. **正向激励 ≥ 反向惩罚**：好行为获得流量倾斜，差行为损失流量。
3. **梯度处罚**：不再"一刀切"，恶意 vs 失误用不同力度区分，且罚款金额与交易金额挂钩。
4. **弱信号交叉验证**：反作弊依赖复合指纹，宁可漏过不可误杀。

## 1. 评分项核心规则（v1.1 调整项）

### 1.1 基础分与活跃（保持稳定）

- **注册/认证**：注册 +30,000，实名 +6,500，绑卡 +5,000，绑定手机/邮箱各 +4,000（/2,000）。
- **充值**：每币 +40，充值经验上限压低至 **200,000**（防止充值刷分）。
- **活跃奖励**：每日交易 +200，连续交易 +500/天，一键买币 +500/单。
- **大额交易奖励（5 档）**：≥5k(+1k)，≥20k(+3k)，≥50k(+8k)，≥100k(+15k)，≥300k(+30k)。

### 1.2 交易行为（金额 + 笔数）

```text
成功：+100/币 +200/笔
失败：-90,000/币 -9,000/笔（失败处罚 ≥ 3× 成功奖励）
```

### 1.3 时效行为 ⭐ v1.1 优化

**卖家：准时放币率**
基本义务不应成为刷分通道，整体奖励减半：
- ≤ 1 分钟：+500
- 1–5 分钟：+200
- 5–15 分钟：+100
- 15 分钟 – 超时前：0
*(新卖家首 10 笔交易强制快速放币 ≤5 分钟，超时直接 -5,000 经验)*

**买家：准时付款率**
- ≤ 1 分钟：+500
- 1–5 分钟：+200
- 超时（订单关闭）：-2,000

### 1.4 工单结果（金额挂钩分级处罚）🔧 v1.1 优化

**工单结果与经验值处理**：
- **卖家主动放币（无工单）**：+3,000
- **判定发币（卖家被强制发）**：-10,000 - (交易金额 × 50)
- **判定退币（彻底不发）**：-30,000 - (交易金额 × 100)
  *(注：工单处罚设上限 = 当前经验值 × 0.8，留申诉空间)*
- **工单判定取消（双方协商）**：
  - 买家未付款时取消 = 双方 0 分。
  - 买家已付款 + 买家主动取消 = 卖家 0 分。
  - 买家已付款 + **卖家发起取消** = 卖家 -5,000。
  - (若 30 天内 ≥ 3 次协商取消 = 卖家 -10,000/次)

**买家侧工单处罚（新增）**：
- 判定发币：买家 +0
- 判定退币且查实买家未付款/假付款：买家 -30,000
- 判定退币且发现假截图：买家 -80,000 + 标记欺诈
- 买家发起工单被驳回（无效工单）：买家 -5,000

## 2. 停权红线与计分制 🔧 v1.1 优化

永久停权红线从"单纯计次数"改为"计分制"，给粗心但非恶意卖家容错：
1. **工单计分**：判定退币 = 1.0 次，判定发币 = 0.5 次。累计 ≥ 3.0 次触发永久停权。
2. 经验值扣到 ≤ 0 → 永久停权。
3. 反洗钱/反诈骗命中 → 立即停权（独立通道）。

---

## 3. 反作弊维度 (Sybil 检测) ⭐ v1.1 落地优化

采用**复合指纹**（屏幕分辨率+OS+语言+时区+电池容量+存储+网络行为）作为弱信号：

1. **IP 维度（弱信号）**：
   - 已知运营商 NAT IP 段 → 忽略 IP。
   - 住宅/商业 IP 同 IP ≥ 5 账号活跃 → 标记"待核实"，必须交叉命中设备/银行卡才处罚。
   - 同 IP + 同设备指纹 + 互相交易 → 高置信度 Sybil，直接标记。
2. **银行卡维度（强信号）**：
   - 同银行卡绑定 ≥ 2 个活跃账号 → 标记"待核实" + 要求人脸/视频验证。
   - 同银行卡作为**收款账户**在 ≥ 2 个账号出现 → 直接标记 Sybil。
3. **惩罚**：
   - 关联账号组内互相交易 → 经验=0，不计入多样性。
   - 对手方集中度异常（多样性指数 < 0.4）→ 日衰减 -200/天。
   - 异常图谱（闭环交易/极短时买卖）→ 该笔经验=0。

## 4. 时间衰减与双轨制展示 ⭐ v1.1 优化

采用 **4 桶增量维护** (bucket_30d / bucket_90d / bucket_180d / bucket_old)，解决全量重算性能问题。

**非对称时间衰减权重**：
- **正向经验**（好事忘得快）：30天内 1.0 / 30-90天 0.7 / 90-180天 0.4 / 180天+ 0.2。
- **负向经验**（坏事记得久）：90天内 1.0 / 90-365天 0.8 / 365天+ 0.5。

**双轨制与降级滞后**：
- **等级保底**：衰减后经验值 ≥ 当前等级下限×0.7，最多降 1 级。
- **用户展示分**：= max(衰减后实际分, 历史最高分 × 0.85)，平滑用户感知。
- **系统内部实际分**：严格按衰减计算，用于流量分配。
- **降级滞后**：升级即时生效；降级需连续 7 天经验低于当前等级下限才执行（降级前 3 天静默提示）。

## 5. 新人沙箱体系 ⭐ v1.1 优化

**取消对手方硬限制**：改为新人首 7 天订单系统自动**优先匹配** L3+ 卖家（软偏好），无人在线则匹配 L2。

**角色与商户双轨沙箱限额**：
1. **新买家**（限额放宽）：
   - ≤7 天：10,000 元/笔
   - 7–30 天：50,000 元/笔
2. **新卖家**（限额严格）：
   - ≤7 天：5,000 元/笔
   - 7–30 天：20,000 元/笔
3. **认证商家快速通道**：
   - 认证后起步：50,000 元/笔
   - 7 天无工单：200,000 元/笔
   - 30 天无工单：无限制。

## 6. 等级与权益体系 🔧 v1.1 优化

完全保留 2024 版的信用等级阶梯（L1~L5）与计算公式 `信用分 = min(950, max(0, f(经验值)))`。

**明确化的等级权益表**：

| 权益 | L1 | L2 | L3 | L4 | L5 |
|---|---|---|---|---|---|
| **一键买币流量权重** | **0.2×** *(注)* | 1.0× | 1.5× | 3.0× | 6.0× |
| **挂单展示优先级** | 末位 | 普通 | 优先 | 优先 + 高亮 | 顶置 |
| **费率折扣 (设月度上限)** | 标准 | 标准 | **97 折** (封顶 500U) | **95 折** (封顶 2kU) | **9 折** (封顶 5kU) |
| **单笔/日交易上限** | 沙箱 | 5w / 20w | 20w / 100w | 100w / 500w | 无限 |
| **客服响应优先级** | 普通 | 普通 | 优先 | 优先 | VIP |

*(注：新注册 30 天内的 L1 用户享受新人扶持流量，保底 3 单/天；30 天以上仍为 L1 才按 0.2× 执行。)*
