# Voss 资金路由图 v2.0

> 商户可配模式 | 入款 → KYT（写标签+存分数）→ 商户业务池 → 分桶归集 → 风险归集池 → 出款

---

## 一、风控标签判定（所有模式统一）

### KYT 入款风控（一次性）
- 每笔入款调用 KYT 接口（仅一次）
- **分数持久化存储**（绑定每笔入款记录）

### 判定逻辑（目录优先于分数）
- **命中高风险目录** → Severe-冻结（不看分数）
  - 制裁名单（OFAC 等）
  - 暗网市场（Darknet Market）
  - 混币器（Mixer / Tumbler）
  - 被盗资金（Stolen Funds）
  - 勒索软件（Ransomware）
  - 欺诈（Fraud）
  - 恐怖融资（Terrorist Financing）
  - 目录由平台统一维护，商户不可修改
- **未命中目录，按分数判定**
  - 0 ~ 20 → Low
  - 20 ~ 50 → Mid
  - 50 ~ 80 → High
  - ≥ 80 → Severe-可用

### Severe 子分类

| 子分类 | 来源 | 状态 |
|---|---|---|
| **Severe-冻结** | 命中高风险目录 | 完全冻结，仅商户后台提款 |
| **Severe-可用** | 仅分数 ≥80，未命中目录 | 可出款给 Severe 地址 |

## 二、商户模式配置

### 模式总览

| | 标准模式（默认） | 专业模式 | 专家模式 |
|---|---|---|---|
| **桶数** | 2 + Severe | 3 + Severe | 4 + Severe |
| **适合** | 新商户 / 小型商户 | 中等规模商户 | 大型商户 / 有风控团队 |
| **切换** | 默认 | 商户申请 → 平台审批 | 商户申请 → 平台审批 |

### 标准模式 —— 2 桶

| 桶 | 对应 KYT 标签 | 分数范围 |
|---|---|---|
| **干净桶** | Low + Mid + High | 0 ~ 79 |
| **严重桶** | Severe-可用 + Severe-冻结 | ≥ 80 或命中目录 |

- 借款：不需要（非严重只有一个桶）
- 出款 KYA：非 Severe 地址 → 干净桶出；Severe 地址 → 严重桶出

### 专业模式 —— 3 桶

| 桶 | 对应 KYT 标签 | 分数范围 |
|---|---|---|
| **低风险桶** | Low + Mid | 0 ~ 49 |
| **高风险桶** | High | 50 ~ 79 |
| **严重桶** | Severe-可用 + Severe-冻结 | ≥ 80 或命中目录 |

- 借款：低 ↔ 高 双向可借，Severe 不参与
- 出款 KYA：Low/Mid 地址 → 低风险桶优先；High 地址 → 高风险桶优先

### 专家模式 —— 4 桶

| 桶 | 对应 KYT 标签 | 分数范围 |
|---|---|---|
| **低风险桶** | Low | 0 ~ 20 |
| **中风险桶** | Mid | 20 ~ 50 |
| **高风险桶** | High | 50 ~ 80 |
| **严重桶** | Severe-可用 + Severe-冻结 | ≥ 80 或命中目录 |

- 借款优先级：
  - High 不足 → 先借 Mid → 再借 Low
  - Low 不足 → 先借 Mid → 再借 High
  - Mid 不足 → 先借 High → 再借 Low
  - Severe 不参与借款

## 三、资金流程（所有模式通用）

### 1. 入款阶段
- A. **入款检测**：监控区块链，检测入账交易
- B. **交易确认**：等待 N 个确认

### 2. KYT 风控
- C. **一次性 KYT**：调用 KYT 接口，输出风险标签 + 分数，持久化存储

### 3. 商户业务池
- D. **商户业务池入账**：资金进入商户业务池（待归集余额），每笔入款绑定风险标签
- 此阶段不进入风险桶

### 4. 分桶归集
- E. **归集触发**：达到阈值 / 闲置归集 / 手动触发
- E0. **分桶归集计划**：按风险标签 + 当前模式的映射规则，分配到对应桶
- F. **归集交易**：执行链上归集转账，归集阶段不做 KYT

### 5. 出款阶段

#### 常规出款（KYA 判定）
- G. **出款请求**：输入目标地址 + 金额
- H. **KYA 判定**：判定目标地址风险标签（Low / Mid / High / Severe）
- H0. **出款池路由**：按当前模式映射到对应桶，同标签桶优先
- I. **余额检查 & 借款补足**（非 Severe 桶）
  - 优先从同标签桶扣款
  - 不足时按借款规则向其他非 Severe 桶借款

#### Severe 出款

| 场景 | 规则 |
|---|---|
| Severe-可用 → Severe 地址 | ✅ 允许，自动执行，无需审批 |
| Severe-可用 → 非 Severe 地址 | ❌ 不允许 |
| Severe-冻结 → 商户后台提款 | ✅ 允许，可到任意地址，无需审批 |
| 非 Severe 桶 → Severe 地址 | ❌ 不允许 |
| Severe 参与跨桶借款 | ❌ 不允许 |

#### 出款执行
- J. **出款交易**：广播到链上，等待确认

### 6. 结算
- K. **账本更新**：更新资金池余额，写入审计日志

## 四、模式切换

### 切换流程
- 商户发起申请 → 平台审批 → 切换生效

### 历史资金处理（无 KYT 分数）

> 旧系统只区分地址是否脏，KYT 做过但分数未保存

| 已有资金状态 | 归入 |
|---|---|
| 未被隔离（干净地址） | 干净桶 / 低风险桶（视模式） |
| 已被隔离（脏地址） | Severe 桶 |

### 有 KYT 分数的资金（新系统产生）

| 切换方向 | 处理方式 |
|---|---|
| 标准 → 专业 | 干净桶按保存的分数拆分：0~49 → 低风险桶，50~79 → 高风险桶；无分数的全归低风险桶 |
| 标准 → 专家 | 干净桶按分数拆分：0~20 → 低，20~50 → 中，50~80 → 高；无分数的全归低风险桶 |
| 专业 → 专家 | 低风险桶按分数拆分：0~20 → 低，20~49 → 中；高风险桶映射为高；无分数的保留原桶 |
| 专家 → 专业 | 低 + 中 → 低风险桶；高 → 高风险桶 |
| 专家 → 标准 | 低 + 中 + 高 → 干净桶 |
| 专业 → 标准 | 低 + 高 → 干净桶 |
| 所有模式 | Severe 桶不变 |

## 五、核心约束

| # | 约束 | 说明 |
|---|------|------|
| 1 | **KYT 必执行，分数持久化** | 每笔入款调用一次 KYT，分数绑定入款记录永久保存 |
| 2 | **高风险目录优先于分数** | 命中目录 → 强制 Severe-冻结，不看分数 |
| 3 | **高风险目录平台统一** | 商户不可自定义，平台统一维护 |
| 4 | **分数阈值固定** | 20 / 50 / 80 分界线不可调 |
| 5 | **分桶在归集阶段** | 按风险标签 + 模式映射分桶，归集阶段不做 KYT |
| 6 | **禁止跨桶混合** | 入款/归集/记账不跨桶 |
| 7 | **出款 KYA 同标签优先** | 优先从与目标地址同标签的桶出款 |
| 8 | **借款仅非 Severe 桶** | 借款用于出款流动性，不改变入款归属标签 |
| 9 | **Severe 不借不贷** | 不参与跨桶借款 |
| 10 | **Severe-冻结完全隔离** | 命中高风险目录的资金冻结，仅商户后台可提 |
| 11 | **Severe-可用仅出 Severe** | 分数高但未命中目录的资金，只能出给 Severe 地址 |
| 12 | **默认标准模式** | 新商户默认标准模式，升级需申请审批 |

## 六、流程总览

```
入款检测 → 交易确认 → 一次性 KYT（写标签 + 存分数）
                              ↓
                    ┌── 命中高风险目录？──┐
                    ↓ 是                  ↓ 否
              Severe-冻结           按分数打标签
                    ↓            (L / M / H / Severe-可用)
                    └──────┬──────────┘
                           ↓
                    商户业务池入账（待归集）
                           ↓
                       归集触发
                           ↓
              分桶归集（按标签 + 模式映射）
              ┌─────┬─────┬─────┐
              ↓     ↓     ↓     ↓
      ┌─────────────────────┐  Severe桶
      │ 按模式分为 2/3/4 桶  │  (冻结+可用)
      └─────────┬───────────┘     │
                ↓                 │
            出款请求 ←────────────┘
           ↙        ↘
    常规出款    商户后台提款(Severe-冻结)
       ↓              ↓
    KYA判定       可到任意地址
       ↓           无需审批
  出款池路由            ↓
 (同标签桶优先)         │
       ↓               │
  余额检查              │
  ↙       ↘             │
足够    不足→借款        │
  ↓    (非Severe)       │
  ↓       ↓             │
  Severe-可用            │
  →Severe地址           │
  (自动,无需审批)        │
       ↓               ↓
     出款交易 ←─────────┘
       ↓
     账本更新
       ↓
   ✓ 出款完成
```
