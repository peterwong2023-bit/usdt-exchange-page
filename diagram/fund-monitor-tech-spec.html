3. API 数据模型与配置参数

3.1 商户热池配置（merchant_hot_pool_config）

每个商户独立配置，在商户编辑页设置。本页面只读消费这些参数。

字段类型含义示例
HOT_MINdecimal最低线，低于此值告警需补给50,000
HOT_TARGETdecimal目标线 = (HOT_MIN + HOT_MAX) / 2150,000
HOT_MAXdecimal最高线，超过此值触发回储300,000
BATCH_MINdecimal最低回储金额（单次回储门槛）50,000
ENERGY_THRESHOLDinteger执行回储所需最低能量65,000
ENERGY_PER_TXinteger每笔 TRC20 Transfer 能量消耗32,000
ENERGY_SAFETYfloat能量安全系数1.2

3.2 实时数据（接口需提供）

字段类型含义
HOT_BALANCEdecimal当前热池链上 USDT 余额
CURRENT_ENERGYinteger当前账户可用能量
today_out_totaldecimal今日出款总额
today_hot_indecimal今日入热池总额

3.3 计算字段（前端推导）

can_store    = HOT_BALANCE - HOT_TARGET           // 可回储金额
batch_need   = max(0, BATCH_MIN - can_store)       // 距 BATCH_MIN 的缺口
tx_cap       = floor(CURRENT_ENERGY / (ENERGY_PER_TX × ENERGY_SAFETY))   // 当前可转笔数
tx_need      = ceil(ENERGY_THRESHOLD / (ENERGY_PER_TX × ENERGY_SAFETY))  // 所需笔数
net_change   = today_hot_in - today_out_total      // 今日净变化（不含回储）

6. 回储任务 — 生命周期、状态机与执行规则

6.1 任务生命周期

系统扫描→ 发现满足条件→ 自动创建任务→ 等待能量→ 能量满足 → 立即执行→ 成功 / 失败
  • 创建时机：系统定时扫描发现余额超目标线且 can_store ≥ BATCH_MIN 时自动创建
  • 执行时机：条件满足后立即执行，不等待下次扫描
  • 核心原则：水位线状态只影响"新任务创建"，不影响已有任务执行

6.2 任务类型

统一为一种类型，不再区分"普通"和"强制"。超过 HOT_MAX 时的差异仅用告警级别体现（页面红色 + Telegram 紧急通知）。
回储金额 = HOT_BALANCE − HOT_TARGET（余额超出目标线的部分）

6.3 任务状态机（4 种状态）

状态内部枚举含义tag 颜色备注列内容
等待能量WAIT_ENERGY任务已创建，能量不足黄色阻塞原因，如 "能量等待 可转 1 笔 / 需 2 笔"

12. 推荐值算法（后续迭代）

本页面将提供 HOT_MIN 和 HOT_MAX 的推荐值，辅助运营在商户编辑页配置。

HOT_MIN 推荐值

Hot_Min = PeakOut_per_hour × H × Safety
参数含义推荐值
PeakOut_per_hour过去 14~30 天每小时出款峰值（取 P95）按实际数据
H热池独立扛住的时长6h / 12h / 24h（推荐 12h）
Safety保险系数1.1 ~ 1.3

直觉：能扛住 H 小时出款尖峰的钱。

HOT_MAX 推荐值

Hot_Max = Hot_Min + k × BATCH_MIN
参数含义推荐值
k允许累积几个"回储批次"的缓冲2 或 3

直觉：安全底线 + 2~3 次回储批次的缓冲，超过就该回储。

HOT_TARGET

Hot_Target = (Hot_Min + Hot_Max) / 2
