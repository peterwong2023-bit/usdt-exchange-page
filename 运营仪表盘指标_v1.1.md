## 1. 北极星指标（1 个数字看全局）

### 「自然撮合率」

```
自然撮合率 = 1 - (工单数 / 总订单数)

目标：
- 当前基线（旧体系）：~ 99.5%
- 上线 30 天：≥ 99.6%
- 上线 90 天：≥ 99.7%
- 长期目标：≥ 99.8%
```

→ **每提升 0.1pp 意味着每天少几百单工单 → 客服工作量直接下降。**

### 2.1 自然撮合率（同上）

### 2.2 AI 自动判定率

```
AI 自动判定率 = AI 高置信度被采纳数 / 总工单数

目标：
- T+30 ≥ 50%
- T+90 ≥ 70%
- T+180 ≥ 85%
```

### 2.3 平均放币时长

```
卖家放币时长 = 买家点已付款 → 卖家放币 的中位数

目标：
- 当前基线：~ 6 min
- 上线 30 天：≤ 5 min
- 上线 90 天：≤ 4 min
- 长期目标：≤ 3 min
```

### 2.4 客服平均处理时长

```
客服处理时长 = 工单创建 → 判定完成 的中位数（不含等待）

目标：
- 当前基线：~ 5 min
- 上线 30 天：≤ 3 min
- 上线 90 天：≤ 2 min
```

## 3. AI 判定效能

### 3.1 AI 建议准确率

```
准确率 = 客服未修改 AI 建议的工单数 / AI 给出建议的工单数

按置信度区间细分：
≥ 95%: 准确率应 ≥ 99%
90–95%: 应 ≥ 95%
80–90%: 应 ≥ 90%
70–80%: 应 ≥ 85%
< 70%: 不强制（人工主导）
```

### 3.2 AI 建议采纳率

```
采纳率 = 客服一键确认数 / AI 给出建议数

目标：≥ 70%（说明 AI 建议有用）
```

### 3.3 申诉率（事后验证）

```
申诉率 = 用户对 AI 判定提出申诉的占比
申诉成立率 = 申诉被推翻的占比

目标：
- 申诉率 ≤ 1%
- 申诉成立率 ≤ 5%（即 95% 申诉是无理的）
```

### 3.4 AI 判定 vs 人工判定一致性

```
取一致样本（AI 建议 = 客服最终决定）vs 不一致样本
分析不一致的特征,反向训练
```

### 3.5 OCR / 多模态识别准确率

```
OCR 字段提取准确率（金额/时间/收款方/交易号 各自统计）
目标：≥ 99%

OCR 识别错误的工单 → 必须人工介入
```

## 4. 信用体系健康度

### 4.1 等级分布

```
                  T+0     T+30    T+90
L5 信用极好        2.1%    2.3%    2.8%
L4 信用优秀       12.5%   13.8%   15.2%
L3 信用良好       38.2%   38.9%   39.5%
L2 信用中等       38.7%   37.2%   35.1%
L1 信用较差        8.5%    7.8%    7.4%
```

→ 健康趋势：L4/L5 缓慢上升,L1 下降,L3 稳定。

### 4.2 等级流转矩阵

```
            升级数  降级数  净变化
L1→L2        1,200    -      +1,200
L2→L3        2,400   320    +2,080
L3→L4          580   180     +400
L4→L5           80    20      +60
```

→ 看是否有"虚假升级"（升得太快）或"过度降级"（规则太严）。

### 4.3 经验值通胀率

```
全平台经验值总量 / 用户数 的趋势

如果快速上涨 → 可能存在刷分漏洞
如果快速下跌 → 可能规则过严
```

### 4.4 准时放币率分布

```
卖家准时放币率分布（直方图）：
≥ 95%: 65%
85-95%: 20%
70-85%: 10%
< 70%: 5%
```

→ 跟踪低分段是否在缩小。

### 4.5 工单失败永久停权数

```
每日新增永久停权用户数（工单 3 次触发）
目标：稳定在低位，不应突然激增
```

## 5. 撮合质量

### 5.1 一键买币成功率

```
一键买币成功率 = 撮合成功 / 一键买币请求

目标：≥ 95%
```

### 5.2 一键买币首次匹配时长

```
首次匹配时长 = 请求发起 → 卖家响应 的中位数

目标：≤ 30 sec
```

### 5.3 流量基尼系数（GINI）

```
卖家撮合量分布的不平等度
0 = 完全平均, 1 = 完全集中

目标：0.4–0.6（合理倾斜,但不极端）
> 0.7：警告（流量过度集中）
```

### 5.4 商户视角"流量公平性"

```
分等级看撮合次数 / 在线时长

L5：~ 8 单/小时
L4：~ 6 单/小时
L3：~ 4 单/小时
L2：~ 2 单/小时
L1：~ 0.5 单/小时
```

→ 应呈现明显但不悬殊的阶梯。

### 5.5 撮合多样性

```
近 30 天每个商户的"独立买家数 / 总订单数"
> 0.7：健康
0.4–0.7：偏集中
< 0.4：高度集中（可能是 Sybil 或老熟客圈）
```

### 5.6 高峰期 vs 低谷期对比

```
凌晨 0-6 点 vs 晚上 19-22 点：
- 在线商户数
- 撮合成功率
- 平均放币时长
- 工单率
```

→ 凌晨表现差是常态,但**不应该比上线前更差**。

## 6. 风险监控

### 6.1 Sybil 关联组

```
新发现的 Sybil 关联组数（每日）
关联组活跃度
关联组涉及金额
被自动降权的组数
```

### 6.2 异常交易图谱

```
检测到的闭环交易（A→B→C→A）数
可疑短时反复交易数
被标记的"互刷"账号数
```

### 6.3 截图复用检测

```
付款截图哈希查重命中率
→ 命中后必须人工介入
```

### 6.4 收款方不匹配率

```
卖家预绑定账户 vs 实际收款方 不一致比例
异常率突涨 → 可能是规则配置问题
```

### 6.5 可疑账号告警

```
新注册 7 天内交易额 > 5 万：警告
单设备多账号活跃 ≥ 3：警告
单 IP 多账号活跃 ≥ 5：警告
```

### 6.6 工单率异常商户 Top N

```
近 7 天工单率 > 5% 的商户列表
→ 自动降级为 L1 或转人工审核
```

## 7. 用户分层透视

### 7.1 沉默 vs 活跃

```
近 30 天活跃用户 / 总注册用户
近 30 天交易过的用户 / 总注册用户
```

### 7.2 不同等级的留存

```
L5 商户 30 天留存率
L4 商户 30 天留存率
...
L1 商户 30 天留存率
```

### 7.3 新商户成长漏斗

```
注册 → 实名 → 绑卡 → 首单 → 沙箱毕业（L2）

每一步流失率
```

### 7.4 流失预警

```
原 L4/L5 商户 30 天内突然不活跃 → 重点关注
近 7 天交易量下降 ≥ 50% 的商户 → 主动联系
```

## 8. 告警事件

### 8.1 自动告警规则

| 触发条件 | 告警级别 | 通知对象 |
|---|---|---|
| 自然撮合率单日下跌 > 0.3pp | P0 | 全员 |
| AI 建议准确率单日下跌 > 5pp | P1 | 算法团队 |
| 平均放币时长上升 > 30sec | P1 | 撮合团队 |
| 工单数单日激增 > 50% | P0 | 全员 |
| Sybil 组单日新增 > 100 | P1 | 风控团队 |
| 永久停权 > 50 / 日 | P1 | 风控团队 |
| 客服处理时长上升 > 20% | P2 | 客服团队 |
| 流量 GINI > 0.7 | P2 | 撮合团队 |

### 8.2 告警响应 SLA

| 级别 | 响应时间 | 处理时间 |
|---|---|---|
| P0 | 5 分钟 | 1 小时 |
| P1 | 30 分钟 | 4 小时 |
| P2 | 4 小时 | 24 小时 |
