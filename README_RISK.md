# 商户级风控打分与熔断系统 (Withdraw)

## 核心设计
本系统专门针对 **Withdraw (出款)** 业务设计，旨在防范黑客攻击、内鬼风险及系统异常。

### 1. 风险三态管理
- **正常 (NORMAL)**: 分数 < 40，业务无限制。
- **预警 (WARN)**: 40 ≤ 分数 < 100，仅记录并提示，业务不限制。
- **熔断 (FUSE)**: 分数 ≥ 100，**立即停止出款 (STOP_WITHDRAW = ON)**。

### 2. 六大触发规则
| 规则 | 权重 | 说明 |
|---|---|---|
| 单笔金额异常 | 1.0 | 超过 P99 分位数 |
| 累计金额异常 | 0.9 | 10m/1h 累计金额超过基准 |
| 出款频率异常 | 0.8 | 1m/10m/1h 频率过高 |
| 小额高频 | 0.8 | 占比高且持续天数多 |
| 机械节奏 | 0.6 | 出款时间间隔极其规律 (stddev 小) |
| 地址集中度 | 0.7 | 资金流向过于集中 (Top1/Top2 占比高) |

### 3. 分数衰减与解封
- **衰减**: 
  - WARN 状态：每 30 分钟 -20 分，降至 40 以下转为 NORMAL。
  - FUSE 状态：每 30 分钟 -10 分，最低降至 50，**不自动解封**。
- **解封策略 A**: 
  - 手动执行策略 A。
  - 操作：停止出款 OFF，分数设为 40，状态转为 WARN。

## 页面说明
- `MerchantRiskMonitoringPage.html`: 实时监控大盘，支持详情查看、规则命中分析及一键执行解封策略 A。
- `MerchantRiskRulesMock.ts`: 提供符合最新文档要求的模拟数据结构。
